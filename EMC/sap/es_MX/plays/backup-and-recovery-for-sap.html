Optimización de aplicaciones existentes

Respaldo y recuperación para SAP

  • Las bases de datos de SAP de varios terabytes obligan a muchos clientes a clonar el almacenamiento del nivel 1, descargarlo en cintas y administrar restauraciones incrementales. Este ambiente es ideal para BRS de última generación de EMC con deduplicación.

Los tres retos principales

  1. Presiones sobre las ventanas de respaldo de SAP

    Las bases de datos de SAP de varios terabytes obligan a muchas empresas a clonar al almacenamiento costoso de nivel 1 para agilizar el respaldo. Además, esta estrategia de clonación presenta un riesgo, ya que la falta de respaldos completos requiere la coordinación de restauraciones incrementales. Por último, se requiere tiempo para localizar las cintas almacenadas fuera del sitio y llevarlas al sitio.

    Data Domain es ideal para superar este reto; Callaway Golf obtiene tasas de deduplicación de 25:1 para SAP, con un rendimiento de más de 1 TB por hora, lo cual permite más respaldos a disco completos para lograr una restauración sencilla.

  2. Costo: el espacio físico de los datos de SAP incluye grandes sistemas no productivos.

    Los ambientes no productivos de SAP, que incluyen el de recuperación de desastres, soporte de producción, sistema de garantía de calidad, pruebas, desarrollo y capacitación, consumen más almacenamiento que el de producción y también necesitan protección. Si el ambiente de producción ocupa 3 TB, un solo respaldo puede ser de 12 TB (de 3 a 4 veces el tamaño del ambiente de producción).

    Los beneficios del respaldo deduplicado se extienden a los ambientes no productivos, lo cual incluye los respaldos completos de todo el ambiente SAP y recuperaciones basadas en disco más rápidas.

  3. Los administradores de SAP y de bases de datos necesitan visibilidad y control de administración.

    Para Oracle: SAP tiene herramientas para administrar el respaldo y la recuperación. Hay tres herramientas de software certificadas con SAP BRtools para Oracle, lo que permite que los administradores de SAP administren los respaldos integrados en software de respaldo centralizado. Respaldo de otros fabricantes

    NetWorker se integra en API específicas de los proveedores, lo cual incluye SAP BRtools para base de datos Oracle; de esta manera, se elimina la necesidad de desarrollar y mantener scripts personalizados.

El valor de EMC

  • La deduplicación de EMC Data Domain es una mejor práctica para el respaldo y la recuperación de SAP.

    Las bases de datos de producción de SAP de varios terabytes son muy comunes y crecen de manera exponencial cuando se incluyen los ambientes no productivos (pruebas, desarrollo, sistema de garantía de calidad y recuperación de desastres). Resultados de costo y complejidad: a) es posible que se use almacenamiento de nivel 1 y clonaciones para enfrentar las ventanas de respaldo, b) los respaldos incrementales generan ventanas de recuperación complejas y extensas, y c) los ambientes no productivos multiplican tres o cuatro veces la cantidad de datos de respaldo.

    Punto de prueba: Callaway Golf obtiene tasas de deduplicación de 25:1 con un rendimiento de más de 1 TB/h en producción, lo que se traduce en más respaldos completos. Al ejecutar un ambiente sin cintas, se eliminan costos de administración.

  • Administración unificada de la base de SAP llevada a cabo por administradores de respaldo y de bases de datos.

    La integración de NetWorker en SAP y SAP BRtools para la base de datos de Oracle proporciona una visibilidad transparente y administración unificada a los administradores de bases de datos y de SAP sin la necesidad de desarrollar y mantener scripts personalizados.

    Punto de prueba: Sports Chalet depende de la deduplicación de EMC Avamar con EMC NetWorker para unificar la administración de respaldo y recuperación. Con Avamar, los respaldos de SAP se completan en 20 minutos en lugar de 16 horas. Con NetWorker, se requieren de 5 a 10 minutos por día para la administración del respaldo, en lugar de las horas que se necesitaban anteriormente.

    NOTA: Para NetWorker con SAP HANA, consulte el escenario de ventas de HANA.

  • Simplificación de la recuperación de desastres para SAP.

    Algunos clientes de SAP quizá busquen una solución de recuperación de desastres de menor costo para distancias asíncronas con estrategias de reinicio sin preparación. Data Domain les permite a los clientes de SAP ampliar sus respaldos locales para obtener una recuperación de desastres eficiente y escalable, con tecnologías de replicación que reducen los requisitos de ancho de banda hasta en un 99 % cuando se transfieren datos a distancia.

    Punto de prueba: Callaway Golf usa el respaldo deduplicado de Data Domain para facilitar la recuperación de desastres y replica sus datos de respaldo desde San Diego hasta Carlsbad (California) a un menor costo, ya que reutilizan el ancho de banda de WAN para otros procesos importantes del negocio.

La competencia

Lo que posiciona la competencia

IBM posiciona Tivoli Storage Manager (TSM) para administración de datos de SAP, lo cual incluye respaldo y recuperación. TSM está certificado para SAP BRtools para Oracle PERO NO para SAP HANA.

Respuesta de EMC

EMC NetWorker también es compatible con DB2 y Oracle DBMS for SAP y está certificado con SAP BRtools para Oracle. Aproveche sus recursos técnicos de EMC para proporcionar referencias detalladas de las diferencias competitivas y de historias de éxito.

Lo que posiciona la competencia

Symantec NetBackup está certificado con SAP BRtools para bases de datos de Oracle y SAP HANA. Symantec puede intentar aprovechar su certificación para HANA, lo cual es posible que no sea importante para muchos usuarios de SAP actualmente.

Respuesta de EMC

EMC NetWorker está certificado con SAP BRtools para Oracle pero NO para SAP HANA. Aproveche sus recursos técnicos de EMC para proporcionar referencias detalladas de las diferencias competitivas y de historias de éxito.

Preguntas de calificación

  • TI: ¿Cómo están reduciendo los costos para los respaldos de SAP? ¿Cuentan con la flexibilidad de realizar respaldos completos diarios?

    Generalmente, los clientes crean clones en el almacenamiento de nivel 1 con snapshots incrementales. Es posible que los respaldos completos se reserven solo para los fines de semana. La cinta se usa para el largo plazo. Analice enfoques alternativos con la posibilidad de respaldos completos diarios.

    Incorpore en la conversación la deduplicación, la manera en que el cliente puede aumentar la cantidad de respaldos completos y la posibilidad de almacenarlos de forma diaria en un dispositivo especialmente diseñado para respaldo de bajo costo. Punto de prueba: Intente compartir los casos de Callaway Golf, Sports Chalet, Aramis y otros clientes de SAP.

  • TI: ¿Ejecutan SAP en Oracle? ¿Y usan SAP BRtools?

    Si el cliente ejecuta SAP en Oracle, la integración en BRtools puede ser un requisito para administrar el respaldo basado en Oracle para SAP.

    Explique que NetWorker se integra en API específicas de los proveedores, lo cual incluye SAP BRtools para Oracle, y que, de esta manera, se elimina la necesidad de desarrollar y mantener scripts personalizados.

  • TI: ¿Cuál es su estrategia de recuperación de desastres para SAP? ¿Qué tienen implementado actualmente?

    La recuperación de desastres será importante para el cliente, aunque, con frecuencia, no se implemente bien. Busque clientes de SAP que dependan de la cinta para la recuperación de desastres. Es posible que algunos clientes de SAP no tengan un plan debido al costo, particularmente, en el caso de distancias asíncronas.

    Punto de prueba: Intente compartir la implementación de Data Domain llevada a cabo por Callaway Golf.

Manejo de objeciones

  • Las bases de datos de SAP cuentan con altas tasas de cambio y logs de archivo/transacciones, por lo cual no son adecuadas para la deduplicación.

    Es cierto que muchos métodos de deduplicación no satisfacen los requisitos de este ambiente. Pero la deduplicación de longitud variable de Data Domain permite obtener tasas de deduplicación líderes del sector en el área de respaldo de bases de datos. Los clientes con tasas de cambio diarias del 10 % al 15 % y retención de 4 a 6 semanas logran tasas de deduplicación de 7 a 10 veces.

    Callaway Golf, que es un cliente de SAP con una base de datos de SAP de varios terabytes, obtiene tasas de deduplicación de 25:1 en su sitio primario y su sitio de recuperación de desastres.

  • Mis administradores de SAP y de bases de datos desean tener visibilidad directa. No quieren depender de los administradores de respaldo.

    NetWorker les proporciona a los administradores de SAP y de bases de datos control y visibilidad directos. NetWorker también se integra en API específicas de los proveedores y SAP BRtools para bases de datos de Oracle, lo cual elimina la necesidad de scripts personalizados.

    Para Data Domain, la utilería nativa de base de datos (Oracle RMAN, volcados de SQL, etc.) les da control a los administradores de bases de datos. Los administradores de bases de datos escriben en un sistema Data Domain que actúa como una plataforma de almacenamiento de protección consolidada para toda la empresa.
