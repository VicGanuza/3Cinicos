HANA listo para el centro de datos

Soluciones para SAP HANA

  • HANA es una tecnología revolucionaria que genera conversaciones nuevas en las cuentas de SAP sobre analítica en tiempo real y la nueva plataforma en la memoria para el conjunto de aplicaciones de negocios, BW y data marts de SAP. HANA requiere una nueva infraestructura.

Los tres retos principales

  1. Identificación del uso del negocio para HANA

    SAP HANA es una “tecnología revolucionaria” que requiere una gran inversión para adoptarla e implementarla. Para obtener la aprobación de la inversión y el presupuesto, generalmente, los propietarios de las aplicaciones deben ejecutar una prueba de concepto con un caso de uso que demuestre su valor y el ROI.

    Ayude al cliente a identificar un partner que pueda definir un caso de uso con él. Trabaje con el líder de GTM de EMC para SAP en su región a fin de obtener asistencia para buscar partners.

  2. Garantía de alta disponibilidad y protección de datos

    SAP HANA está madurando. Los clientes quieren trabajar con partners que puedan brindar orientación y soluciones para proteger datos, garantizar la alta disponibilidad y minimizar el impacto en el RTO y el RPO en caso de tiempo fuera planificado o no planificado.

    EMC implementó HANA y puede compartir mejores prácticas. Presente los sistemas de respaldo y recuperación, la tolerancia a desastres, la recuperación de desastres sin preparación y las fortalezas históricas que diferencian a EMC para los clientes de SAP.

  3. El dispositivo HANA prediseñado es rígido y costoso

    La estrategia inicial de SAP para la implementación de HANA, como un dispositivo prediseñado y certificado, está resultando un reto para los clientes debido a su estricta lista de materiales y su costo. En su lugar, muchos clientes prefieren aprovechar el almacenamiento existente.

    Informe al cliente que SAP ofrece el programa de integración adaptada del centro de datos (TDI) para aprovechar el almacenamiento existente. El cliente debe ponerse en contacto con su representante de SAP para la asignación.

El valor de EMC

  • Dispositivo prediseñado de escalamiento horizontal de Cisco o VCE

    "EMC se asoció con Cisco y VCE para ofrecer una familia de dispositivos certificados por SAP, creados en servidores blade Cisco UCS y con almacenamiento VNX 5300. El dispositivo “Enterprise” (empresarial) escala a 16 nodos y un dispositivo “Starter” (de inicio) escala hasta 4 nodos, lo que cubre el rango desde SMB y PoC, hasta implementaciones de clase empresarial.

    La API de bloques de SAP, desarrollada junto con EMC, permite la integridad de datos y failovers no disruptivos. Y la plataforma Cisco UCS ofrece alta disponibilidad, administración integral simplificada y corrección automatizada del servidor, almacenamiento y dispositivos de red.

  • Protección de datos y tolerancia a desastres para SAP HANA

    Respaldo y recuperación: Data Domain está disponible para respaldo NFS de HANA y se puede administrar desde HANA Studio. SAP certifica software respaldo de otros fabricantes integrado en BR Tools para HANA. DD Boost está programado para la certificación del trimestre 1. Actualmente, los clientes pueden usar Networker, si desarrollan un script personalizado.

    Tolerancia a desastres: SAP certifica protección síncrona de tolerancia a desastres. EMC ofrece una solución certificada por SAP, con protección sincrónica proporcionada por MirrorView/S.

    Recuperación de desastres: SAP no ha publicado una estrategia de protección asíncrona de HANA, pero Data Domain se puede usar para replicación a larga distancia y recuperación de desastres sin preparación.

  • Integración adaptada del centro de datos (TDI) para HANA

    Para reducir los costos de hardware y operaciones, SAP HANA ahora se puede usar con la plataforma de almacenamiento preferida del cliente. Esto permite a EMC posicionar a VMAZ, cuando el cliente no desea usar el modelo de dispositivo.

    DI alcanzó la GA a fines de 2013, y MAX KJ, 20K y 40K son plataformas de almacenamiento certificadas para TDI. Comprometa a los clientes interesados de forma anticipada para aprovechar y preservar el espacio físico de almacenamiento de EMC.

    Los candidatos ideales incluyen: 1) el cliente de EMC VMAX que desee aprovechar el almacenamiento existente y 2) los clientes comprometidos con Cisco, VCE o IBM para servidores de HANA.

La competencia

Lo que posiciona la competencia

IBM incluye al giro comercial para definir el caso de uso de HANA con las soluciones de implementación rápida de SAP, consiguiendo así probar el valor de HANA, un requisito previo para la inversión.

IBM ofrece dos tipos de dispositivo: Una solución de nodo único para implementaciones menores y otra de escalamiento horizontal con GPFS. GPFS aprovecha los discos locales y no necesita arreglo externo.

Para la tolerancia a desastres, IBM posiciona GPFS certificado de SAP.

Respuesta de EMC

HANA es una conversación estratégica. Contacte con los propietarios de aplicaciones y el departamento de TI para identificar y validar los casos de uso. Aproveche el SI o partner local que trabaja con el cliente.

GPFS agrega un nivel de complejidad más alto al ambiente HANA y requiere un conjunto de habilidades adicionales.

Si el cliente NO acepta el modelo del dispositivo o NO desea considerar un servidor de Cisco, considere presentar TDI* para proteger el almacenamiento. Considere lo que implica ser partner de Cisco. Posicione Data Domain para BRS. * El cliente debe solicitar soporte de TDI al representante de SAP.

Lo que posiciona la competencia

HP posiciona soluciones de implementación rápida para SAP (RDS), lo cual permite al giro comercial obtener logros rápidos probando el valor de HANA; este es un requisito previo para las inversiones en HANA. HP ofrece tres configuraciones de dispositivo: una de nodo único para menor volumen de datos con almacenamiento interno y dos configuraciones de escalamiento horizontal (la basada en EOL EVA e IBRIX y un modelo nuevo basado en el almacenamiento 3PAR). HP está certificado por SAP para tolerancia a desastres.

Respuesta de EMC

Solicite la participación de los propietarios de aplicaciones y TI. Identifique un caso de uso y obtenga un logro rápido. Trabaje con el SI o partner local para impulsar las conversaciones y la POC. Tenga cuidado con las ofertas de HP de EVA/IBRIX. Tiene un costo menor que la 3PAR pero repercute en el rendimiento. La solución 3PAR se ofreció para elevar HP hasta las capacidades de rendimiento de EMC, pero tiene un costo más alto. Si el cliente NO acepta el modelo del dispositivo o NO desea considerar un servidor de Cisco, considere presentar TDI* para proteger el almacenamiento. Considere lo que implica ser partner de Cisco. Posicione Data Domain para BRS. * El cliente debe solicitar soporte de TDI al representante de SAP.

Lo que posiciona la competencia

El dispositivo de escalamiento horizontal para HANA de Cisco con NetApp tiene la misma configuración que EMC, con la excepción del back-end de almacenamiento, que es de NetApp. La solución NetApp aprovecha NFS para el almacenamiento compartido.

Como HANA está en la memoria, el almacenamiento pretende sobre todo conseguir que los datos de SAP persistan. El posicionamiento competitivo se centra en el componente del servidor (en este caso, de Cisco) y cómo se agrupa con otros proveedores de servidores.

NetApp intentará que Cisco acceda a las cuentas de EMC.

Respuesta de EMC

EMC debe recordar que NetApp intentará introducirse en las oportunidades de Cisco y EMC. Es importante trabajar en la planificación de la cuenta y las relaciones con sus colegas de Cisco.

Note la naturaleza de NFS como sistema de archivos compartidos y conozca los problemas técnicos con NFS y la disponibilidad. EMC desarrolló la API de bloques de SAP junto con SAP, que es hacia donde la mayoría de los proveedores se están mudando.

Si EMC necesita proponer la integración adaptada del centro de datos (TDI), considere el impacto, ya que Cisco puede presentar a NetApp.

Lo que posiciona la competencia

Fujitsu se centra en los equipos de TI con servicios de planificación, creación y administración para HANA.

Fujitsu ofrece dos configuraciones de dispositivo: una configuración de un solo nodo para pequeños volúmenes de datos con almacenamiento interno para brindar persistencia y una configuración de escalamiento horizontal con almacenamiento unificado de NetApp.

Fujitsu posiciona tolerancia a desastres para SAP y respaldo y recuperación integrados.

Respuesta de EMC

Para el dispositivo HANA, VCE y Cisco ofrecen dispositivos para competir con Fujitsu.

Note la naturaleza existente de NFS como sistema de archivos compartidos. EMC desarrolló la API de bloques de SAP junto con SAP para eliminar los problemas asociados con NFS y HANA.

Es importante para EMC enfocarse en los propietarios de aplicaciones y en TI para que identifiquen un caso de uso y permitir un logro rápido para probar el valor de HANA. Trabaje con el SI o el partner local para impulsar estos debates y estos logros rápidos.

Preguntas de calificación

  • TI: ¿Desean implementar SAP HANA? ¿Han desarrollado una estrategia de consumo y un plan para enfrentar la alta disponibilidad, la protección de datos y la recuperación de desastres?

    Debido a que, en la arquitectura HANA, el almacenamiento se usa para persistencia de datos, la disponibilidad y la protección de datos serán los diferenciadores clave. A medida que las empresas quieran adoptar HANA, buscarán partners que puedan ayudarlos a ocuparse de la protección de datos, la alta disponibilidad y la continuidad del negocio con esta plataforma en la memoria. Use esto como oportunidad para hablar de la trayectoria de EMC y de sus fortalezas.

  • TI de aplicaciones: ¿Están considerando SAP HANA para mejorar la analítica en tiempo real de SAP y la visibilidad de los datos de SAP? ¿Han definido un caso de uso para SAP HANA?

    Las interrupciones de HANA y los gastos obligaron a los clientes a desarrollar un caso de uso para probar su valor antes de la adopción. Póngase en contacto con el SI o el partner local para desarrollar un caso de uso con el cliente. Busque trabajar con Cisco o VCE para el dispositivo que se usará para la prueba de concepto.

  • Gerentes séniors de TI: Su empresa, ¿está considerando implementar HANA como plataforma estratégica para SAP? ¿Estaría interesado en analizar enfoques para garantizar que HANA esté “listo para el centro de datos” a fin de proteger los sistemas y los datos?

    Nuevamente, debido a que, en la arquitectura HANA, el almacenamiento se usa para persistencia de datos, la disponibilidad y la protección de datos serán los diferenciadores clave. Los ejecutivos valoran las asociaciones para HANA que proporcionan ofertas de clase empresarial. Identifique los casos de uso planificados para HANA (data mart, BW en HANA o conjunto de aplicaciones empresariales). Comprenda con quién están trabajando.

    Cree valor: enfatice que EMC usa HANA, tiene soluciones de clase empresarial probadas y puede compartir sus experiencias y mejores prácticas.

Manejo de objeciones

  • SAP HANA es un sistema en la memoria con almacenamiento exclusivo para persistencia. ¿Cuál es el valor que proporciona el almacenamiento?

    Recuerde al cliente que SAP es de misión crítica para el negocio. Si los servidores fallan, se pierden los datos en la memoria. SAP ahora proporciona HANA para ERP y el conjunto de aplicaciones empresariales. Las empresas piden y deberían esperar los mismos niveles de alta disponibilidad, protección de datos y continuidad del negocio con HANA que con los modelos tradicionales de DBMS.

    EMC ha trabajado junto con SAP para desarrollar mejores prácticas y soluciones para proteger los ambientes SAP y los datos de SAP a fin de permitir SLA óptimos para el RPO y el RTO. Esto continúa con HANA.

  • Somos un cliente de EMC y queremos aprovechar nuestro VMAX existente, pero VMAX no está certificado con un dispositivo de un proveedor de servidores.

    SAP ahora ofrece la posibilidad de ejecutar servidores SAP HANA con el almacenamiento preferido por el cliente para reducir los costos operacionales y de hardware en la base instalada de clientes mediante la reutilización de los componentes de hardware y los procesos operacionales existentes. La propuesta de SAP es la integración adaptada del centro de datos (TDI).

    Actualmente, SAP está llevando a cabo varios proyectos piloto para la TDI de SAP HANA. Los clientes pueden asignarse mediante sus respectivos representantes de cuentas de SAP para este programa.
