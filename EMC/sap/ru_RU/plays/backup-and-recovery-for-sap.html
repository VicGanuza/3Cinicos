Оптимизация устаревших приложений

Резервное копирование и восстановление для SAP

  • Использование баз данных SAP, объемы которых измеряются терабайтами, приводит к тому, что многим заказчикам приходится клонировать данные в системы хранения уровня 1, выгружать их на ленты и управлять инкрементными восстановлениями. Эта среда идеально подходит для решений следующего поколения EMC BRS с функцией дедупликации.

Три основные проблемы

  1. Необходимость уменьшения окон резервного копирования для SAP

    Использование баз данных SAP, объемы которых измеряются терабайтами, приводит к тому, что многим предприятиям для ускорения резервного копирования приходится клонировать данные в дорогостоящую систему хранения уровня 1. Кроме того, эта стратегия клонирования несет с собой риск, поскольку недостаточное количество полных резервных копий требует координации инкрементных восстановлений. И наконец, для поиска и доставки лент, хранящихся на удаленных площадках, необходимо время.

    DataDomain отлично подходит для таких случаев. Предприятие Callaway Golf достигло коэффициента дедупликации 25:1 для SAP при производительности более 1 ТБ/ч, что позволяет создавать больше полных резервных копий дисков и обеспечить простое восстановление.

  2. Стоимость: в SAP хранятся данные непроизводственных систем большого объема

    Непроизводственные среды SAP, включающие средства аварийного восстановления, поддержки производства, систему контроля качества, средства тестирования, разработки и обучения, используют больший объем пространства в системах хранения, чем производство, и их тоже необходимо защищать. Если для производственных целей необходимо 3 ТБ данных, то одна резервная копия может иметь размер 12 ТБ (в 3—4 раза больше, чем необходимо для производства).

    Преимущества дедуплицированного резервного копирования выходят за рамки непроизводственных сред и включают полное резервное копирование всего ландшафта SAP и быстрое восстановление на основе дисков.

  3. Администраторам SAP и баз данных необходима наглядность и средства управления

    Для Oracle: в SAP имеются средства для управления резервным копированием и восстановлением. Три программных продукта сертифицированы с помощью SAP BRTools для Oracle, что позволяет администраторам SAP управлять резервными копиями с помощью средств, интегрированных в централизованное ПО для резервного копирования. Средства резервного копирования сторонних поставщиков

    Продукт NetWorker интегрирован с API конкретных поставщиков, включая SAP BRtools для баз данных Oracle, благодаря чему нет необходимости разрабатывать и поддерживать пользовательские скрипты.

Ценностное предложение EMC

  • Дедупликация EMC Data Domain — это передовая практика для резервного копирования и восстановления SAP.

    Производственные базы данных SAP, объемы которых измеряются терабайтами, — обычное дело. Размеры таких баз данных увеличиваются в геометрической прогрессии, если к ним добавить непроизводственные среды (среды тестирования, разработки, систем контроля качества и аварийного восстановления). Дороговизна и сложность приводят к следующим результатам: а) для решения проблемы окон резервного копирования используются системы хранения уровня 1 и клонирование; б) из-за использования инкрементных резервных копий процесс восстановления становится долгим и сложным; в) непроизводственные среды увеличивают объем данных резервных копий в 3 или 4 раза.

    Аргументы. Предприятие Callaway Golf достигло коэффициента дедупликации 25:1 при производительности более 1 ТБ/ч для производственных систем, что позволяет создавать больше полных резервных копий. Теперь исключены расходы на управление ленточной средой.

  • Унифицированное управление и администрирование базовыми системами SAP для администраторов баз данных и резервного копирования

    Интеграция NetWorker с SAP и SAP BRtools или базами данных Oracle обеспечивает полную наглядность и унифицированное управление для администраторов баз данных и SAP без необходимости разрабатывать и поддерживать пользовательские скрипты.

    Аргументы. Предприятие Sports Chalet использует дедупликацию EMC Avamar с EMC NetWorker для унификации управления восстановлением резервных копий. Благодаря Avamar резервное копирование SAP выполняется за 20 минут, а не за 16 часов. Благодаря NetWorker на администрирование резервного копирования уходит всего лишь 5—10 минут в день, а не несколько часов, как раньше.

    ПРИМЕЧАНИЕ. Сведения об использовании NetWorker с SAP HANA см. в сценарии HANA.

  • Упрощение аварийного восстановления для SAP

    Некоторым заказчикам SAP может потребоваться менее дорогостоящее решение для аварийного восстановления, предназначенное для асинхронных расстояний и использующее стратегию «холодного» перезапуска. Data Domain позволяет заказчикам SAP расширять локальные резервные копии для эффективного и масштабируемого аварийного восстановления с помощью технологий репликации, на 99% уменьшающих требования к полосе пропускания при перемещении данных на расстояние.

    Аргументы. Для упрощения аварийного восстановления предприятие Callaway Golf использует дедуплицированное резервное копирование Data Domain, реплицируя данные резервных копий из Сан-Диего в Карлсбад, Калифорния, по сниженной стоимости благодаря повторному использованию пропускной способности глобальной сети для других критически важных бизнес-процессов.

Конкуренция

Предложение конкурента

Корпорация IBM предлагает Tivoli Storage Manager (TSM) для управления данными SAP, в том числе для резервного копирования и восстановления. Продукт TSM сертифицирован с помощью SAP BRtools для Oracle, НО НЕ с помощью SAP HANA.

Ответ EMC

Продукт EMC NetWorker также поддерживает DB2 и СУБД Oracle для SAP и сертифицирован с помощью SAP BRtools для Oracle. Используйте технические ресурсы EMC для обеспечения уверенной конкурентной дифференциации и достижения успеха.

Предложение конкурента

Продукт Symantec NetBackup сертифицирован с помощью SAP BRtools для Oracle и баз данных SAP HANA. Symantec может попытаться использовать имеющуюся у нее сертификацию HANA, которая сегодня уже не так важна для многих пользователей SAP.

Ответ EMC

Продукт EMC NetWorker сертифицирован с помощью SAP BRtools для Oracle, НО НЕ с помощью SAP HANA. Используйте технические ресурсы EMC для обеспечения уверенной конкурентной дифференциации и достижения успеха.

Вопросы для выявления потенциальных клиентов

  • ИТ. Как вы уменьшаете расходы на резервное копирование SAP? Используете ли вы достаточно гибкие инструменты, чтобы выполнять ежедневное полное резервное копирование?

    Обычно заказчики создают клоны в системах хранения уровня 1 с помощью инкрементных моментальных снимков. Полное резервное копирование может выполняться только в выходные дни. Для долгосрочного хранения данных используется лента. Обсудите альтернативные подходы, обеспечивающие возможность ежедневного полного резервного копирования.

    Начните обсуждение с дедупликации и с того, как заказчик может увеличить количество полных резервных копий с возможностью ежедневного сохранения их на экономичных специализированных устройствах для резервного копирования. Аргументы. Предоставьте информацию о предприятиях Callaway Golf, Sports Chalet, Aramis и других заказчиках SAP.

  • ИТ. Запускаете ли вы SAP на Oracle и используете ли SAP BRtools?

    Если заказчик использует SAP на Oracle, то интеграция с BRtools может быть одним из требований для управления резервным копированием для SAP на основе Oracle.

    Расскажите о том, что NetWorker интегрируется с API поставщиков, а также о том, что благодаря использованию SAP BRtools для Oracle нет необходимости разрабатывать и поддерживать пользовательские скрипты.

  • ИТ. Какова ваша стратегия аварийного восстановления для SAP? Что вы используете в настоящее время?

    Функции аварийного восстановления важны для пользователей, хотя зачастую они плохо реализованы. Ищите заказчиков SAP, использующих ленту для аварийного восстановления. Некоторые заказчики SAP могут не иметь плана из-за затрат, особенно для асинхронных дистанций.

    Аргументы. Предоставьте информацию о развертывании Data Domain в Callaway Golf.

Ответы на возражения

  • Базы данных SAP быстро изменяются, а наличие журналов архивирования и операций затрудняет дедупликацию.

    В этой среде многие способы дедупликации не работают. Но в Data Domain используется дедупликация блоков переменной длины, которая обеспечивает ведущий в отрасли коэффициент дедупликации резервных копий баз данных. Заказчики, у которых ежедневный процент изменения данных составляет 10—15% и которым требуется срок хранения 4—6 недель, уменьшат объем данных в 7—10 раз.

    Предприятие Callaway Golf — заказчик SAP с базой данных SAP, объем которого измеряется терабайтами — достигло коэффициента дедупликации 25:1 на своей основной площадке и площадке аварийного восстановления.

  • Мои администраторы SAP и баз данных требуют непосредственной наглядности: они не хотят полагаться на средства администрирования резервного копирования.

    NetWorker предоставляет администраторам SAP и баз данных непосредственный контроль и наглядность. NetWorker также интегрируется с API конкретных поставщиков, включая SAP BRtools для баз данных Oracle, благодаря чему нет необходимости разрабатывать и поддерживать пользовательские скрипты.

    Встроенные в Data Domain утилиты для работы с базами данных (Oracle RMAN, дампы SQL и т. д.) предоставляют полноценный контроль администраторам баз данных. Администраторы баз данных могут выполнять запись в систему Data Domain, которая служит консолидированной платформой защиты данных для всего предприятия.
