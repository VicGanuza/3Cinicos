Продукт HANA, подготовленный к использованию в центрах обработки данных

Решения для SAP HANA

  • HANA — это «подрывная технология», вызывающая в среде заказчиков SAP новые обсуждения, посвященные аналитике в реальном времени и новой платформе, работающей в памяти, для SAP Business Suite, BW и витрин данных. Для HANA требуется новая инфраструктура.

Три основные проблемы

  1. Определение способов использования HANA в бизнесе

    SAP HANA — это «подрывная технология», для внедрения и развертывания которой необходимы значительные инвестиции. Чтобы получить одобрение подразделений, отвечающих за бюджет и инвестиции, владельцам приложений обычно необходимо провести тестовое испытание в условиях сценария использования, чтобы продемонстрировать преимущества и окупаемость инвестиций.

    Помогите заказчикам найти партнера, который сможет помочь определить сценарий использования. Работайте с местным контактным лицом EMC, занимающимся выводом SAP на рынок, чтобы получить помощь в поиске партнеров.

  2. Гарантия высокой доступности и защиты данных

    SAP HANA вступает в период зрелости. Заказчики ищут партнеров, которые могут дать рекомендации и решения для защиты данных, гарантировать высокую доступность и минимизировать влияние на целевое время восстановления и целевую точку восстановления в случае запланированных или незапланированных простоев.

    EMC внедрила HANA и может поделиться передовыми практиками. Расскажите о решениях BRS корпорации EMC, устойчивости к авариям, «холодном» аварийном восстановлении и исторически общепризнанных сильных сторонах EMC для заказчиков SAP.

  3. Готовые устройства HANA — негибкие и дорогостоящие

    Изначальная стратегия SAP по внедрению HANA в виде сертифицированного готового устройства создает проблемы для заказчиков из-за своей стоимости и негибких характеристик. Многие заказчики предпочитают использовать вместо этого существующие системы хранения.

    Проинформируйте заказчиков о том, что SAP предлагает программу адаптированной интеграции с центрами обработки данных (TDI), предназначенную для использования имеющихся систем хранения. Чтобы участвовать в этой программе, заказчикам необходимо связаться со своим представителем SAP.

Ценностное предложение EMC

  • Готовое устройство для горизонтального масштабирования от Cisco или VCE

    Корпорация EMC заключила партнерское соглашение с Cisco и VCE, чтобы предложить семейство устройств с сертификацией для SAP, построенных на основе блейд-серверов Cisco UCS и системы хранения VNX 5300. Устройство уровня «Enterprise» масштабируется до 16 узлов, а «Starter» — до 4, что позволяет удовлетворить все потребности как тестировочных сред и сред малых предприятий, так и развертываний корпоративного класса.

    Блочный программный интерфейс SAP Block API, разработанный EMC совместно с партнерами, обеспечивает целостность данных и предоставляет возможность аварийного переключения без прерывания работы. Кроме того, платформа Cisco UCS включает функции высокой доступности, упрощенного комплексного управления и автоматического восстановления серверов, систем хранения и сетевых устройств.

  • Защита данных и устойчивость к авариям для SAP HANA

    Продукты для резервного копирования и восстановления: Система Data Domain доступна для резервного копирования HANA с помощью NFS и ею можно управлять из HANA Studio. SAP сертифицирует ПО для резервного копирования сторонних поставщиков, интегрированное в BRTools для HANA. Сертификация DD Boost запланирована на первый квартал. В настоящее время заказчики могут использовать Networker, если разработают собственный сценарий.

    Устойчивость к авариям. SAP сертифицирует синхронную защиту для обеспечения устойчивости к авариям. EMC предлагает сертифицированное SAP решение с функциями синхронной защиты на основе MirrorView/S.

    Аварийное восстановление: SAP не опубликовала стратегию асинхронной защиты HANA, но можно использовать Data Domain для удаленной репликации и «холодного» аварийного восстановления.

  • Адаптированная интеграция с центрами обработки данных (TDI) для HANA

    Чтобы сократить затраты на оборудование и операции, SAP HANA теперь можно использовать с платформой хранения заказчика. Это позволяет EMC предлагать систему VMAX, если заказчик не хочет внедрять аппаратную модель.

    Возможности интеграции с центрами обработки данных стали доступны в конце 2013 года, а платформы хранения VMAX 10K, 20K и 40K были сертифицированы для TDI. Привлекайте заказчиков на ранних этапах, чтобы задействовать и закрепить положение EMC на рынке систем хранения.

    Идеальный кандидат должен обладать следующими характеристиками: 1) заказчик EMC VMAX, который хочет использовать существующие системы хранения; 2) заказчик, который использует серверы Cisco, VCE или IBM для HANA.

Конкуренция

Предложение конкурента

IBM привлекает бизнес-подразделения для определения сценариев использования HANA.

IBM предлагает две конфигурации устройств: одноузловое решение для малых развертываний и горизонтально масштабируемое решение с GPFS. GPFS работает с локальными дисками.

Для обеспечения устойчивости к авариям IBM предлагает продукт GPFS, сертифицированный SAP.

Ответ EMC

HANA — это в первую очередь стратегия. Привлекайте владельцев приложений для определения сценариев использования. Используйте системных интеграторов.

GPFS усложняет среду HANA.

Если модель с использованием устройства НЕ подходит или пользователь НЕ будет использовать сервер Cisco, расскажите о TDI* для защиты системы хранения. Расскажите о партнерстве с Cisco и о Data Domain. * Необходимо запрашивать поддержку TDI у SAP

Предложение конкурента

Решения для быстрого развертывания SAP (RDS), позволяющие добиться быстрых успехов. Доступно три конфигурации устройств: одноузловая для небольших объемов и две горизонтально масштабируемых. Первая основана на EOL EVA (IBRIX еще предлагается), а новая — на 3PAR. Решения сертифицированы SAP на предмет устойчивости к авариям.

Ответ EMC

Привлекайте владельцев приложений. Определяйте сценарии использования. Работайте совместно с системным интегратором. Решение HP EVA/IBRIX дешевле, чем 3PAR, но имеет меньшую производительность. 3PAR предлагается как решение HP с производительностью EMC, но стоит дороже. Если устройство НЕ подходит или пользователь НЕ будет использовать сервер Cisco, расскажите о TDI* для защиты СХД. Расскажите о партнерстве с Cisco и Data Domain. * Необходимо запрашивать поддержку TDI у SAP

Предложение конкурента

Горизонтально масштабируемое устройство Cisco для HANA с NetApp имеет такую же конфигурацию, как EMC, за исключением серверной части. NetApp использует NFS в качестве общей системы хранения.

СХД используется для обеспечения устойчивости данных SAP. Конкурентная позиция основана на серверном компоненте (Cisco).

NetApp будет искать возможность повлиять на Cisco и получить доступ к заказчикам EMC.

Ответ EMC

Важно проработать и спланировать отношения с заказчиком совместно с Cisco.

Функциональность NFS уже устарела и может создать технические проблемы. EMC разработала программный интерфейс SAP Block API совместно с SAP, и теперь на него переходят большинство поставщиков.

Если необходимо предложить программу интеграции с ЦОД (TDI), рассмотрите возможный ущерб, поскольку Cisco может начать предлагать NetApp.

Предложение конкурента

Fujitsu концентрируется на ИТ-командах, занимающихся HANA.

Fujitsu предлагает две конфигурации устройств: одноузловую для небольших объемов и горизонтально масштабируемую с унифицированной СХД NetApp.

Предлагается средства обеспечения устойчивости к авариям для SAP, интегрированного резервного копирования и восстановления.

Ответ EMC

Устройства для HANA VCE и Cisco способны конкурировать с продуктами Fujitsu.

Функциональность NFS в качестве общедоступной файловой системы устарела. EMC разработала программный интерфейс SAP Block API совместно с SAP, чтобы решить проблемы, которые возникают с NFS и HANA.

Важно сконцентрироваться на владельцах приложений и ИТ-специалистах, помогая им определять сценарии использования. Работайте с системным интегратором, чтобы добиться быстрых успехов.

Вопросы для выявления потенциальных клиентов

  • ИТ. Планируете выполнить развертывание SAP HANA? Разработали ли вы стратегию использования ресурсов и план для обеспечения высокой доступности, защиты данных и аварийного восстановления?

    Поскольку системы хранения в архитектуре HANA используются для обеспечения устойчивости данных, доступность и защита данных будет нашим ключевым дифференциатором. Если компании хотят внедрить HANA, они будут искать партнеров, которые смогут помочь решить вопросы защиты данных, высокой доступности и непрерывности бизнеса для данной платформы, работающей в памяти. Используйте это как предпосылку для информирования об истории EMC и ее ключевых сильных сторонах.

  • ИТ-специалисты по приложениям. Вам необходимо, чтобы SAP HANA предоставляла расширенную аналитику SAP и обеспечивала наглядность данных SAP в реальном времени? Определили ли вы сценарий использования для SAP HANA?

    Благодаря тому, что HANA — «подрывная технология» и для ее внедрения необходимы определенные расходы, перед внедрением HANA заказчикам требуется разработать сценарий использования для извлечения максимальной пользы. Привлекайте системных интеграторов и локальных партнеров, которые могут создать сценарии использования совместно с заказчиком. Попробуйте поработать с Cisco или VCE, чтобы использовать устройство в тестовом испытании.

  • Старшие ИТ-руководители. Рассматривает ли ваша компания HANA в качестве стратегической платформы для SAP? Заинтересованы ли вы в изучении подходов, гарантирующих готовность HANA для использования в центрах обработки данных и защиты систем и данных?

    Поскольку системы хранения в архитектуре HANA используются для обеспечения устойчивости данных, доступность и защита данных будет нашим ключевым дифференциатором. Руководители ценят партнеров, которые могут сформировать предложения корпоративного класса для HANA. Продемонстрируйте запланированные сценарии использования для HANA: витрина данных, BW на HANA или Business Suite. Постарайтесь понять, с кем работают заказчики.

    Создайте преимущества: расскажите о том, что EMC использует HANA, имеет испытанные решения корпоративного класса и может поделиться опытом и передовыми практиками.

Ответы на возражения

  • SAP HANA работает в памяти, а системы хранения используются только для обеспечения устойчивости. Какие преимущества предоставляют системы хранения?

    Напомните заказчикам, что система SAP критически важна для их бизнеса. Если серверы выйдут из строя, то данные, хранящиеся в памяти, будут утеряны. Теперь SAP делает HANA доступной для ERP и Business Suite. Компании требуют и ожидают тот же самый уровень высокой доступности, защиты данных и непрерывности бизнеса при использовании HANA, что и при использовании традиционных моделей СУБД.

    Корпорация EMC совместно с SAP разработала передовые практики и решения для защиты ландшафтов SAP и данных, чтобы обеспечить оптимальные соглашения об уровне обслуживания для целевой точки восстановления и целевого времени восстановления. Это справедливо и для HANA.

  • Мы — заказчики EMC и хотим использовать имеющуюся у нас систему VMAX, но она не сертифицирована для работы с устройствами наших поставщиков серверов.

    Теперь SAP предоставляет возможность запускать сервер SAP HANA совместно с используемой заказчиком системой хранения, чтобы уменьшить расходы на оборудование и эксплуатацию благодаря использованию имеющегося у заказчиков оборудования и производственных процессов. SAP призывает использовать адаптированную интеграцию с центром обработки данных (TDI).

    В настоящее время SAP проводит с заказчиками несколько опытных проектов по использованию SAP HANA TDI. Чтобы участвовать в программе, заказчики должны получить одобрение соответствующего представителя по работе с заказчиками SAP.
