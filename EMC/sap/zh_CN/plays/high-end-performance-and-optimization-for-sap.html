传统应用程序优化

SAP 的高端性能和优化

  • 推动更多 SAP 系统和数据需求的催化剂包括虚拟化、合并和收购。FAST VP 用智能自动存储分层取代了专用分层。

三大挑战

  1. 确保 SAP 高峰性能和高可用性

    当 SAP 性能下降时，会影响数千名用户。在 SAP 环境中，存储是维护性能的重要组件。客户必须为计划的高峰性能调整和保留存储，因而会增加存储成本并降低资源利用率。

    现在，VMAX 利用 FAST VP 带来大量节省，并将性能保证提升到更高水平。证据： 关于客户在降低成本的同时提高性能的 SAP 成功案例。

  2. 因 SAP 增长而导致基础架构需求增加

    SAP 客户承受着纵向和横向扩展基础架构的压力。扩展、收购和 SAP 虚拟化是主要驱动因素。一个客户一年内将存储基础架构扩展了一倍。另一个客户由于 SAP 增长，而用尽了数据中心空间。

    VMAX 拥有骄人的成绩，因为该平台可以满足纵向扩展和横向扩展需求。根据 IDC 调查，EMC 是 SAP 客户的首选存储。有关更多背景信息，请参阅销售资源中的客户成功案例。

  3. 生产效率下降： 在 SAP 环境中，需要手动操作和干预

    存储、SAP Basis 和数据库管理员花费大量时间手动监视和管理 SAP IT。此外，SAP Basis 还缺少支持拷贝和更新 SAP 系统和数据的端到端自动化解决方案，特别是用于启动 SAP 的克隆后步骤。

    FAST VP 消除了手动操作，确保存储和性能优化。SAP LVM 与 Timefinder 相集成，提供自动化 SAP 克隆和系统更新。

EMC 价值

  • 利用全面的自动化存储分层降低成本并提高性能。

    现在，许多客户根据预估周期性高峰工作负载（例如，月末/季度末的订单处理需求）采用专用 FC 存储来支持生产。结果是： 降低了高峰工作负载性能并导致资源利用率低下。

    由于手动密集操作和缺少 SAP 验证，客户迟迟未能采用分层存储。FAST VP 打破了这一采用障碍。

    证据： Callaway Golf 采用适用于 SAP 的 FAST VP。调查位于行动资源部分。结果证实，降低了成本并提高了性能。

  • 通过扩展虚拟 SAP 环境减少了物理占用空间

    FAST VP 减少了物理存储占用空间，同时又应对了增长的虚拟 SAP 系统数量。

    证据： Kindred Healthcare 是一家医疗保健服务公司，在 46 个州拥有 7.6 万名员工，近年来扩张步伐迅猛并且收购了其他公司，因而给 IT 基础架构带来了巨大压力。虚拟化的副作用，他们的存储基础架构在 1 年内扩大了一倍。

    通过基于 EMC 和 VMWare 技术实现虚拟化，Kindred 的性能提高了 300%，并且将 SAP 的交付周期从 45 天缩短到几个小时。FAST VP 通过减少物理占用空间，每年帮助节省了 100 万美元的运营支出。

  • 针对 TST、DEV 和 QAS 自动执行 SAP 系统克隆、拷贝和数据更新，降低了成本

    SAP 团队为了支持开发、测试和培训，需要多 TB 生产数据库的多个拷贝。

    EMC 存储与 SAP Landscape Virtualization Manager (LVM) 相集成，支持 SAP 管理员使用 EMC 存储复制软件，自动执行系统克隆、拷贝和数据更新。借助 EMC Snap On Snap 技术，客户可以降低与创建拷贝以及更新系统和数据相关联的成本并提高效率。

竞争

竞争对手推介

NetApp 的 SAP 战略以容易争取的客户为目标，重点关注非生产系统、测试、开发和培训。作为战略的一部分，他们将与 SI 合作，进军 SAP 项目。NetApp 与 SAP Landscape Virtualization Manager 相集成，并且推介可节省成本的快照技术。利用 EMC 技术资源提供详细的竞争优势。

EMC 的回应

推介 EMC 在 SAP 客户中拥有的强大存储实力（据 IDC 调查）。对于生产环境，介绍采用 FAST VP 的 EMC 客户案例调查，进一步证实推介。对于非生产环境，推介如何扩展 VMAX，重点介绍自动化和保持较小的数据占用空间；EMC 与 SAP LVM 相集成、分层存储和快照功能。

竞争对手推介

在运行 AIX/DB2 的 SAP 客户中，IBM 将是最具挑战性的对手，因为他们会推介在适用于 SAP 的 AIX /DB2 之间强大的 SVC 集成。如果客户使用 SAP LVM，IBM 的 Tivoli 可以提供存储级别集成，同时其 PowerVM 可以实现服务器级别集成。IBM 将推介与 EMC 类似的核心功能。利用 EMC 技术资源提供详细的优于 SVC 的竞争优势。

EMC 的回应

推介 EMC 在 SAP 客户中拥有的强大存储实力（据 IDC 调查）。SAP 努力开发仅在 x86 上运行的 HANA，因而，可以推介 EMC 在更多方面与 SAP 战略保持一致，因为我们开放支持 x86 和 UNIX。利用 SAP 客户 VMAX 案例研究从价值方面进行竞争。利用 EMC 技术资源提供详细的 VMAX 优于 SVC 的竞争优势。

资格确认问题

  • 存储/SAP IT： 您在手动监视和管理性能上花费多少时间？在高峰需求期间（例如： 月末或季度末），您是否担心会出现状况？

    许多 SAP 客户调整存储并专门使用第 1 层存储来处理一般在月末和季度末出现的高峰工作负载。虽然这样会减少出现性能瓶颈的风险，但是会浪费资源并提高存储成本。此外，监视和管理 SAP 性能也会造成 SAP 管理员和数据库管理员人手紧张。介绍 VMAX FAST VP 基于策略的自动化，SAP 管理员可以确定何时需要访问哪些最重要的数据；并且存储分层可以自动调整自身。

  • VP/IT 总监： 您如何在降低 SAP 基础架构成本的同时确保 SLA？

    推介 EMC 在满足任务关键型 SAP 应用程序（例如，月末和季度末的高峰处理期间）的性能要求方面的领导地位。推介具有变革意义的 FAST VP，实现了自动使用所有可用存储层的功能。因而，降低了存储和运营成本，与此同时也消除了手动干预成本和风险。

    证据： Callaway Golf 和 Kindred Healthcare 降低了资本支出和运营支出，同时还提高了 SAP 的性能。

  • VP/IT 总监： 考虑到未来的合并或收购等业务活动，您认为今后每年的存储增长率会有何变化？

    推介 EMC VMAX 是最佳存储平台，可以为横向和纵向扩展 SAP 虚拟环境提供支持。通过客户证据阐述这一点。例如，Kindred Healthcare Services 的性能提高了 300%，同时将 SAP 部署的交付周期从 45 天缩短到几个小时。此外，FAST 还帮助将每年的运营支出降低了 100 万美元，并且减少了物理占用空间。

    证据： Callaway Golf 和 Kindred Healthcare。位于行动销售资产中的客户幻灯片。

异议处理

  • 确保 SAP 生产环境的性能至关重要，我不太习惯让存储自动管理 SAP 数据分层。

    首先，分享 EMC 的 SAP 客户通过采用 FAST VP 取得了卓越的成果，在降低资本支出和运营支出（介绍 Callaway Golf 的情况）的同时，提高了性能！SAP 管理员可以基于策略确定何时需要访问哪些最重要的数据；存储分层自动调整自身。向客户展示自动化分层如何更好地利用资源、降低成本并提高性能。

  • VMAX 非常适合 SAP 生产，可以确保性能和高可用性，但是我可以使用比较便宜的存储进行测试、培训和开发。

    对于 SAP 非生产环境，推介 EMC 存储如何与 SAP Landscape Virtualization Manager (LVM) 相集成，通过支持 SAP 管理员使用 VMAX Timefinder 软件在较低层次的存储中自动执行系统克隆、拷贝和数据更新，从而提高生产效率。

    推介使用存储分层和 EMC Snap On Snap 技术，有助于降低与创建拷贝以及更新 SAP 系统和数据关联的成本，并提高效率。
