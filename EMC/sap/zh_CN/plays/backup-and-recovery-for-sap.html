传统应用程序优化

SAP 的备份和恢复

  • 多 TB SAP 数据库迫使许多客户克隆第 1 层存储并分载到磁带，从而管理增量恢复。该环境适合采用 EMC 新一代 BRS 以及重复数据消除。

三大挑战

  1. SAP 备份窗口压力

    多 TB SAP 数据库迫使许多公司克隆到昂贵的第 1 层存储，以加快备份。此外，由于缺少完整备份而需要协调增量恢复，因此该克隆战略还会带来风险。最后，查找存储在异地的磁带并运回本地需要花费大量时间。

    DataDomain 是最佳选择；Callaway Golf 在 SAP 环境中实现了 25:1 的重复数据消除率，每小时的性能超过 1 TB，因此支持更加完整的磁盘备份，进而轻松实现恢复。

  2. 成本： SAP 数据占用空间包括大型非生产系统

    SAP 非生产环境包括灾难恢复、生产支持、QAS、测试、开发和培训，不仅消耗比生产更多的存储量，而且还需要保护。如果生产需要 3 TB 容量，那么单个备份需要 12 TB 容量（3 乘以生产大小的 4 倍）。

    经过重复数据消除的备份的好处还可以扩展到非生产环境，包括整个 SAP 环境的完整备份和更加快速的基于磁盘的恢复。

  3. SAP 管理员和数据库管理员 (DBA) 需要可见性和管理控制

    对于 Oracle： SAP 具有用于管理备份和恢复的工具。3 款软件均通过了 SAP BRTools for Oracle 认证，支持 SAP 管理员通过集成的集中式备份软件管理备份。第三方备份

    NetWorker 与特定于供应商的 API 集成，包括适用于 Oracle 数据库的 SAP BRtools，因此不再需要开发和维护自定义脚本。

EMC 价值

  • EMC Data Domain 重复数据消除是适用于 SAP 备份和恢复的最佳做法。

    如果您具有非生产环境（测试、开发、QAS 和灾难恢复），那么多 TB SAP 生产数据库会非常普遍，并且呈指数式增长。成本和复杂性带来的结果是： a) 第 1 层存储和克隆可能用于应对备份窗口；b) 增量备份导致复杂的长期恢复窗口；以及 c) 非生产环境需要的容量为备份数据量的 3 或 4 倍。

    证据： Callaway Golf 实现了 25:1 的重复数据消除率，并且每小时的生产性能超过于 1 TB，因而可以执行更多完整备份。现在运行无磁带环境，消除了管理成本。

  • 针对 SAP Basis 的统一运营管理和事务管理由 DBA 和备份管理员负责

    NetWorker 与 SAP 以及 SAP BRtools for Oracle 数据库的集成为 DBA 和 SAP 管理员提供了无缝可见性和统一管理，而无需开发和维护自定义脚本。

    证据： Sports Chalet 依赖 EMC Avamar 重复数据消除和 EMC NetWorker 统一了备份和恢复管理。借助 Avamar，20 分钟内便可完成 SAP 备份，以往则需要 16 个小时。此外，通过 NetWorker，他们每天花费在备份管理上的时间只有 5-10 分钟，而过去需要几个小时。

    注意： 有关将 NetWorker 与 SAP HANA 结合使用的信息，请参阅 HANA 行动。

  • 加快针对 SAP 的灾难恢复

    一些 SAP 客户可能希望采用冷重启战略来降低远距离异步灾难恢复解决方案的成本。Data Domain 支持 SAP 客户扩展本地备份，在远距离移动数据时，借助复制技术将带宽需求减少 99%，以实现高效可扩展的灾难恢复。

    证据： Callaway Golf 需要将备份数据从圣地亚哥复制到卡尔斯巴德（两个地方都位于加利福尼亚州），他们使用 Data Domain 经过重复数据消除的备份加快了灾难恢复，同时他们重复使用 WAN 带宽执行其他关键业务流程，因此降低了成本。

资格确认问题

  • IT： 您如何降低 SAP 备份的成本？您是否可以灵活地执行每日完整备份？

    一般而言，客户通过增量快照在第 1 层存储创建克隆。完整备份可能只保留在周末进行。将长期使用磁带。讨论有可能执行每日完整备份的其他方式。

    讨论重复数据消除以及客户可以如何增加完整备份数量，有可能每日在低成本专用备份应用装置上存储这些备份。证据： 分享 Callaway Golf、Sports Chalet、Aramis 和其他 SAP 客户案例。

  • IT： 是否基于 Oracle 运行 SAP？是否使用 SAP BRtools？

    如果客户基于 Oracle 运行 SAP，那么可能需要集成到 BRtools，才能管理基于 Oracle 的 SAP 备份。

    讨论 NetWorker 与特定于供应商的 API 集成，包括 SAP BRtools for Oracle，因此不再需要开发和维护自定义脚本。

  • IT： 您的 SAP 灾难恢复战略是什么？您目前采用哪些方式？

    尽管通常并没有很好地实施，但是灾难恢复对于客户非常重要。寻找依赖磁带从灾难执行恢复的 SAP 客户。一些 SAP 客户可能出于成本考虑而未制定计划，尤其是远距离异步恢复。

    证据： 分享 Callaway Golf 如何部署 Data Domain。

异议处理

  • SAP 数据库更改率高，并且包含不太适合进行重复数据消除的归档/事务日志。

    许多重复数据消除方法无法满足该环境中的需求。但是，Data Domain 可变长度重复数据消除的重复数据消除率在数据库备份领域处于业界领先地位。如果客户的每日更改率是 10-15% 且保留时间为 4-6 周，则可以获得 1/10-1/7 的重复数据消除率。

    Callaway Golf 是一个具有多 TB SAP 数据库的 SAP 客户，在主站点和灾难恢复站点实现了 25:1 的重复数据消除率。

  • 我的 SAP 管理员和 DBA 需要直接可见性，他们不希望依赖备份管理。

    NetWorker 为 SAP 管理员和 DBA 提供了直接控制和可见性。NetWorker 还与特定于供应商的 API 以及适用于 Oracle 数据库的 SAP BRtools 实现了集成，不再需要自定义脚本。

    对于 Data Domain，本机数据库应用工具（Oracle RMAN、SQL 转储等）为 DBA 提供了控制权。然后，DBA 可以向 Data Domain 系统写入数据，该系统将用作整个企业的整合保护存储平台。
