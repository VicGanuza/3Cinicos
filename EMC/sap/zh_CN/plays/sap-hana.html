数据中心就绪 HANA

适用于 SAP HANA 的解决方案

  • HANA 是一项颠覆性技术，在 SAP 客户中引发了关于适用于 SAP 业务套件、BW 和数据集市的实时分析和全新内存中平台的新一轮讨论。HANA 需要新基础架构。

三大挑战

  1. 确定 HANA 的业务用途

    SAP HANA 是一项颠覆性技术，采用和部署都需要投入高额资金。要获得预算和投资批准，应用程序所有者通常必须借助展示价值和 ROI 的使用情形来执行概念证明。

    帮助客户确定可以与客户一起定义使用情形的合作伙伴。与您所在区域的 EMC SAP GTM 主管合作，帮助寻找合作伙伴。

  2. 确保高可用性和数据保护

    SAP HANA 日趋成熟。客户希望找到这样的合作伙伴：可以提供数据保护指导和解决方案，在发生计划内或计划外宕机时，能够确保高可用性并最大限度地降低对恢复时间目标 (RTO)/恢复点目标 (RPO) 的影响。

    EMC 实施了 HANA，并且可以分享最佳做法。介绍 EMC 的 BRS、容灾能力、冷冻灾难恢复，以及 EMC 长期以来为 SAP 客户带来的标志性优势。

  3. 预先构建的 HANA 应用装置不仅僵化，而且非常昂贵

    SAP 最初推出的 HANA 实施战略是一个经过认证的预先构建的应用装置，但是事实证明，由于其僵化的 BOM 和高昂成本，而给客户构成了挑战。相反，许多客户都更加倾向于利用现有存储。

    告诉客户 SAP 提供了旨在利用现有存储的定制的数据中心集成 (TDI) 计划。客户应该联系 SAP 代表，了解更多信息。

EMC 价值

  • 来自 Cisco 或 VCE 的预先构建的横向扩展应用装置

    EMC 与 Cisco 和 VCE 合作提供了一系列基于 Cisco UCS 刀片服务器和 VNX 5300 存储的 SAP 认证应用装置。Enterprise 版应用装置可以扩展到 16 个节点，Starter 版应用装置可以扩展到 4 个节点，涵盖从 SMB 和 PoC 到企业级部署在内的所有环境。

    由 EMC 参与开发的 SAP 数据块 API 支持数据完整性和无中断故障切换。而且，Cisco UCS 平台具备高可用性、经过简化的端到端管理以及自动修复服务器、存储和网络设备。

  • 针对 SAP HANA 的数据保护和容灾功能

    备份和恢复： Data Domain 可用于通过 NFS 备份 HANA，并且可以从 HANA Studio 进行管理。SAP 认证集成到 BR Tools 的第三方备份软件，以用于管理 HANA。DD Boost 计划进行第 1 季度认证。目前，如果客户开发了自定义脚本，可以使用 Networker。

    容灾： SAP 认证了 DT 同步保护。EMC 提供 SAP 认证解决方案，其中通过 MirrorView/S 提供同步保护。

    灾难恢复： SAP 尚未发布 HANA 异步保护战略，但是 Data Domain 可用于远距离复制和冷冻灾难恢复。

  • 面向 HANA 的定制的数据中心集成 (TDI)

    为了降低硬件和运营成本，现在可以将 SAP HANA 与客户的首选存储平台配合使用。当客户不愿使用应用装置模式时，这样有助于 EMC 推介 VMAX。

    TDI 于 2013 年年末全面上市，并且 VMAX 10K、20K 和 40K 是经过认证的适用于 TDI 的存储平台。提前接洽感兴趣的客户，利用并保留客户现有的 EMC 存储。

    理想候选对象包括： 1) 想要利用现有存储的 EMC VMAX 客户以及 2) 决定在 HANA 服务器中采用 Cisco、VCE 或 IBM 产品的客户。

竞争

竞争对手推介

IBM 支持业务线 (LOB) 部门定义在 SAP 快速部署解决方案 (RDS) 中利用 HANA 的使用情形，快速地证明 HANA 价值是投资的前提条件。

IBM 提供了两种应用装置： 适合小规模部署的单节点解决方案以及利用 GPFS 的横向扩展解决方案。GPFS 利用本地磁盘并且不需要外部阵列。

从容灾 (DT) 角度来看，IBM 会推介经 SAP 认证的 GPFS。

EMC 的回应

HANA 是一个战略讨论。与应用程序所有者和 IT 人员进行接洽，确定和验证使用情形。利用与客户有合作关系的 SI 或当地合作伙伴。

GPFS 加重了 HANA 环境的复杂性并且需要额外的管理技能组合。

如果应用装置模式不适用，或者客户不考虑 Cisco 服务器，可考虑介绍 TDI* 来保护存储。如果与 Cisco 合作，请考虑潜在影响。推介适用于 BRS 的 Data Domain。*客户必须向 SAP 代表申请 TDI 支持

竞争对手推介

HP 推介 SAP 快速部署解决方案 (RDS)，支持 LOB 快速地证明 HANA 价值是投资的前提条件。HP 提供了三种应用装置配置： 适用于小型数据卷且采用内部存储的单节点配置以及两种横向扩展配置。一种基于已停售的 EVA 和 IBRIX 的旧模式仍在提供，还提供一种基于 3PAR 存储的新模式。HP 经 SAP 认证，可提供容灾 (DT) 功能。

EMC 的回应

与应用程序所有者和 IT 人员进行接洽。确定使用情形并快速取得成功。与 SI 或当地合作伙伴合作，推动讨论和概念证明 (POC)。警惕 HP 会提供 EVA/IBRIX 解决方案。其成本低于 3PAR，而且提升了性能。之所以提供 3PAR 解决方案，是为了将 HP 性能功能提升到 EMC 的水平，但是成本更高。如果应用装置模式不适用，或者客户不考虑 Cisco 服务器，可考虑介绍 TDI* 来保护存储。如果与 Cisco 合作，请考虑潜在影响。推介适用于 BRS 的 Data Domain。*客户必须向 SAP 代表申请 TDI 支持

竞争对手推介

Cisco 与 NetApp 联手推出的 HANA 横向扩展应用装置具有相同配置，只是 EMC 不是存储后端，而采用了 NetApp 存储。NetApp 解决方案利用 NFS 作为共享存储。

由于 HANA 位于内存中，因此存储主要是为了持续提供 SAP 数据。竞争对手推介主要关注服务器组件（在此情形下是 Cisco），以及如何优于其他服务器供应商。

NetApp 将会试图影响 Cisco，让 Cisco 接触 EMC 客户。

EMC 的回应

EMC 必须警惕，NetApp 会试图插手 Cisco/EMC 机会。一定要与 Cisco 相关人员巩固合作关系，制定客户计划。

请注意，NFS 作为共享文件系统的本来性质，以及关于 NFS 和可用性的已知技术问题。EMC 与 SAP 联手开发了 SAP 数据块 API，现在大多数供应商正转而采用该 API。

如果 EMC 需要转变战略并建议定制的数据中心 (TDI)，则需要考虑影响，因为 Cisco 可能会推荐 NetApp。

竞争对手推介

Fujitsu 的重点对象是计划、构建和管理适用于 HANA 的服务的 IT 团队。

Fujistu 提供了两种应用装置配置：适用于小型数据卷的单节点配置，其中内部存储可实现持续性，以及采用 NetApp 统一存储的横向扩展配置。

Fujitsu 推介适用于 SAP 的容灾 (DT) 以及集成备份和恢复。

EMC 的回应

针对 HANA 应用装置，VCE 和 Cisco 提供了多种应用装置，与 Fujitsu 进行竞争。

请注意，NFS 作为共享文件系统的本来性质。EMC 与 SAP 联手开发了 SAP 数据块 API，消除了与 NFS 和 HANA 关联的问题。

EMC 必须关注应用程序所有者和 IT，帮助他们识别使用情形并快速成功地证明 HANA 价值。与具有合作关系的 SI 或当地合作伙伴合作，推动这些讨论并快速取得成功。

资格确认问题

  • IT： 您是否希望部署 SAP HANA？您是否已经制定了消费战略并计划提供高可用性、数据保护和灾难恢复？

    由于在 HANA 体系结构中，存储用于实现数据持久性，因此可用性和数据保护将是我们的主要竞争优势。随着越来越多的公司选择采用 HANA，他们希望合作伙伴可以帮助他们针对此内存中平台实现数据保护、高可用性和业务连续性。利用这次机会，介绍 EMC 历史和核心优势。

  • 应用程序 IT： 您是否希望 SAP HANA 针对 SAP 数据不断增强 SAP 实时分析和可见性？您是否已经定义了 SAP HANA 的使用情形？

    HANA 中断和费用需要客户开发使用情形，以在采用之前证明价值。接洽可以与客户一起构建使用情形的 SI 或当地合作伙伴。与 Cisco 或 VCE 合作，寻找可用于 POC 的应用装置。

  • 高级 IT 管理： 您的公司是否考虑将 HANA 用作 SAP 的战略平台？您是否有兴趣了解各种方式，确保 HANA 具有“数据中心就绪”优势，可保护系统和数据？

    同样，由于在 HANA 体系结构中，存储用于实现数据持久性，因此可用性和数据保护将是我们的主要竞争优势。高层非常重视可以面向 HANA 提供企业级产品的合作关系。了解适用于 HANA 的计划使用情形 — 数据集市、BW on HANA 和/或业务套件。了解他们的合作伙伴。

    创造价值： 介绍 EMC 使用 HANA 的情况；EMC 具有经验证的企业级解决方案；并且分享我们的经验和最佳做法。

异议处理

  • SAP HANA 位于内存中，并且存储仅用于实现持久性。那么存储提供哪些价值？

    提醒客户 SAP 对于企业而言，是一款任务关键型应用程序。如果服务器宕机，内存中的数据会丢失。SAP 现在面向 ERP 和业务套件提供 HANA。公司需要并且应当期望相同级别的高可用性、数据保护和业务连续性，而且 HANA 适用于传统 DBMS 模式。

    EMC 与 SAP 合作制定保护 SAP 环境和数据的最佳做法和解决方案，为 RPO 和 RTO 提供最佳 SLA。因此，将继续采用 HANA。

  • 我们是 EMC 客户，希望利用现有的 VMAX，但是 VMAX 未通过服务器供应商应用装置认证。

    SAP 现在提供在客户首选存储中运行 SAP HANA 服务器的机会，通过重复使用现有硬件组件和运营流程，减少现有客户群的硬件和运营成本。SAP 为此提出了定制的数据中心集成 (TDI) 这一概念。

    SAP 目前正在针对 SAP HANA TDI 执行多项试验性的客户项目。可以由客户各自的 SAP 客户代表为计划指定客户。
