支持云的基础架构

适用于 SAP 的云解决方案

  • SAP 客户计划迁移到虚拟化 x86 平台。
  • 大多数 SAP 用户使用专有 UNIX 平台。HANA 和 HP-UX 可行性提供了令人信服的转型原因。

三大挑战

  1. 减少 SAP 基础架构占用空间和数据中心成本

    开拓新市场、建立制造厂和收购只是加快数据中心增长和扩张的几个驱动因素。SAP 基础架构是这些资源的主要消费者之一。

    Rompetrol 运行 SAP 应用程序所用的服务器从 675 台减少至 63 台，数据中心管理和环境每年节省 350 万美元。

  2. IT 基础架构成为部署 SAP 的瓶颈

    许多 SAP 客户都面临着 SAP 应用程序急剧增长的问题：增长速度超过了现有数据中心和 IT 基础架构资源的发展速度。部署支持项目的 SAP 环境带来了越来越多的重要里程碑，推迟了计划和完成。

    介绍客户证据： West Pharma 在 90 天内完成 SAP 部署，以往需要 6 个月时间。这是部署为 SAP 预先构建的 Vblock 的直接结果。

  3. 监视和管理 SAP 基础架构所需的时间、资源和成本

    SAP 交付经理和实践主管希望降低成本和风险、提高生产效率并减少员工工作时间、保护和改善当前基础架构环境以及高效地满足业务所有者的需求。

    借助 Vblock，Columbia Sportswear 从孤立的装置（CPU、交换机、阵列）迁移到实现 IT 功能和业务应用程序即服务的环境

EMC 价值

  • 借助 EMC x86 迁移服务和聚合基础架构降低 SAP 基础架构成本

    Oracle 等许多供应商不再支持 Itanium 体系结构，致使许多 SAP 客户迁移到 x86。另外，众所周知的一点是，虚拟化软件在 x86 上具有最高性价比优势，因此随着 SAP 客户迁移到云，许多客户都希望迁移到 x86 并设置最佳平台，从而取得成功。

    当转变 SAP 基础架构时，EMC 进行了服务和 VCE Vblock 系统平台迁移，在所有规划、部署和运营阶段可以作为战略合作伙伴。

    证据： 借助 Vblock，West Pharma 将数据中心占用空间减少 50%，并且减少了支持的硬件数量。

  • 缩短了 SAP 部署时间： 基础架构是推动因素，而不是瓶颈

    工厂集成的 Vblock 简化了收购、部署和运营。通过使用服务配置文件，可以快速部署和扩展 SAP，并且其运营模式支持管理员快于传统环境的速度创建和调配 SAP。

    证据： 长期以来，从硬件到达港口的当天到 West Pharma 开始生产，需要 6 个月的时间；借助 Vblock，SAP 团队在 90 天时间内就启动了供应链管理。

    证据： Columbia Sportswear 能够以业务速度进行扩展，因为预先配置且预先测试的基础架构可以轻松扩展 IT。

  • 获得集成平台优势以及针对 SAP 基础架构的单线支持

    统一基础架构经理提供了跨越世界各地的计算、网络和存储资源适用于 Vblock 基础架构的单点管理。

    证据： West Pharma： “我们过去每周都会接到大量有关硬件故障的报修，现在每个月的接听量已经寥寥无几，因此 IT 可以腾出更多时间来关注业务用户的需求。”

    证据： Columbia Sportswear — Vblock 支持从孤立的设备（如 CPU、交换机和 RAID 阵列）迁移到简化的基础架构，迈入了实现 IT 功能和业务应用程序即服务的时代

竞争

竞争对手推介

转变 SAP 基础架构是一项战略讨论和投资。HP 在 IT 管理人员面前将自己介绍为一个“一站式商店”，可规划、部署和运营 SAP；包括将 SAP 从传统大型机或 Unix 平台迁移到 HP CI 的应用程序服务。

HP 会声称“聚合基础架构”这一说法源自他们提供的 3 种 CI 类型（虚拟、云和应用程序）。

EMC 的回应

与 SI 合作非常重要，因为客户希望最大程度地减少管理的供应商数量。由于 EMC 不提供针对 SAP 的应用程序和 IT 服务，因此合作伙伴战略至关重要。

竞争对手推介

IBM 提供了最强大的 SAP 产品组合，可以影响 SAP 业务和 IT 高层。因此，IBM 可以识别并参与早期规划和令人信服的事件，推动业务和 IT 转型。

IBM 只重视实现 SAP SLA，而将“如何实现”问题丢给 SAP。

IBM 的 CI 主题是集成专业知识，通过预先集成的 PureSystems 推介聚合功能。

EMC 的回应

在竞争 SAP 基础架构业务时，识别令人信服的业务和 IT 事件非常重要。对于 IBM，尤为适用。与 SI 合作。SI 一般可以了解 SAP 计划。将 EMC 基础架构服务和 Vblock 与 SI 相结合，会给 IBM 带来无法逾越的障碍。

邀请客户参观 EBC，了解 EMC IT 的基于 Vblock 的 SAP 案例，赢得高层的认可。

竞争对手推介

NetApp 的战略是与 SI 合作，并推介他们与 SAP 的关系。从 CI 角度来看，NetApp 会攻击 Vblock 系统欠缺灵活性，而且昂贵，因为 Vblock 不支持重复使用现有资产。

EMC 的回应

重点介绍 Vblock 的优势，可以加快 SAP 项目和新实施。尤其是，现场 30 天内就可以完成基础架构部署。

证据： West Pharma： 长期以来，从硬件到达港口的当天至 SAP 投入生产需要 6 个月时间，而借助 Vblock，不到 90 天就可以开始运行 SAP。

资格确认问题

  • IT 管理： 您是否计划了新的 SAP 部署？您是否有兴趣了解聚合基础架构和 EMC 服务可以如何减少部署时间和 SAP 投入运行时间？

    借助聚合基础架构，EMC 可以缩短 SAP 部署时间，从硬件抵达到投入生产的时间间隔从 6 个月减少至不到 90 天。

    由于 VCE 已经预先配置和测试 Vblock，因此我们可以快速推进 SAP 部署，并确信基础架构从一开始就可以提供可靠的性能和可用性。

    介绍包括 EMC IT、West Pharma 和 Columbia Sportswear 在内的证据。

  • IT：您运行的 SAP RISC 服务器是否超过 3 年时间甚至有 5 年之久？

    借助 SAP HANA，x86 是一个面向 SAP 业务套件和 BW 的战略方向。SAP 客户正在从传统 SAP 迁移到 x86。此外，SAP 客户还可以通过迁移到 x86 和虚拟化而受益，包括提高性能和应用程序可用性并降低 TCO。

    介绍 EMC IT 案例和基于 Vblock 部署 SAP。推介 EMC 服务和能力，分享适用于 SAP 的最佳做法经验。

  • IT 管理： 您是否需要降低 SAP 基础架构成本？您是否难以解决涉及多家供应商的 IT 问题？

    通过从 Unix 迁移到虚拟化 x86 平台（如 Vblock），您可以通过基础架构标准化显著降低硬件成本和运营成本。

    Columbia Sportswear 选择基于 Vblock 运行 SAP，将计算、网络、存储、虚拟化和管理技术集成到预先打包的聚合基础架构设备中。

    最终简化了管理并且能够以业务速度进行扩展，因为预先配置且预先测试的基础架构可以轻松地扩展 IT。

异议处理

  • 我怀疑 VCE 是否能够实现 SAP 愿景：私有云支持基础架构。

    重点介绍包括 EMC IT、Columbia Sportswear 和 West Pharma 在内的客户案例。谈论通过预先集成且预先验证的系统如何轻松部署。

    SAP 发现最大限制是无法快速轻松地向用户提供他们所需的功能。在虚拟化聚合基础架构中提供 SAP 环境是满足 SAP 客户快速设置和运行这一重要目标的最佳方式之一。Vblock 系统可以通过预先集成且预先验证的系统减少部署复杂性。EMC 服务可加快部署并减少风险。

  • 目前尚无法基于 x86 虚拟化 SAP 环境，我担心会造成业务中断。

    SAP 与 VMware 合作多年，SAP 致力于全面地利用 VMware 技术，并且 EMC 定义了一个标准虚拟化平台。SAP HANA 只支持 x86 并且是未来的 SAP 平台。

    EMC 全球服务和合作伙伴已经将许多客户迁移到虚拟化环境，通过迁移保持了成功的业务运营。EMC 全面检测了基于 x86 平台的新 SAP 环境，以确保新虚拟化生产系统可以平稳运行。

  • 我正在考虑 Oracle Exadata， 所以不需要 SAP 或 VCE。

    Exadata 和 HANA 是两码事，可解决两种不同的 BI 问题。Exadata 定位为极其快速的数据库设备，用作企业数据仓库，而不是像 HANA 这样的分析加速器。HANA 基于更加简单的行业标准基础架构，可以加速 SAP 数据分析。

    IDC 表示，80% 的 SAP 客户使用 Oracle 的数据库。VCE 是一个专家，对于 SAP 和您而言，是一个更为中立的合作伙伴。目前这是理想选择，采用 SAP HANA 和未来硬件。
