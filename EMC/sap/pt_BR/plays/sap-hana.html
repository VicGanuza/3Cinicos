HANA pronto para datacenter

Soluções para SAP HANA

  • O HANA é uma tecnologia inovadora que desencadeia novas discussões em contas SAP para sistemas de análise da informação em tempo real e uma nova plataforma de memória para o pacote corporativo SAP, BW e data marts. O HANA requer uma infraestrutura nova.

Três principais desafios

  1. Identificar o uso de negócios do HANA

    O SAP HANA é uma tecnologia inovadora que requer um investimento significativo para ser adotada e implementada. Para obter aprovação do investimento e do orçamento, geralmente os usuários dos aplicativos precisam executar uma POC (Proof of Concept, prova de conceito) com um caso de uso que demonstre o valor e o retorno sobre o investimento.

    Ajude o cliente a identificar um parceiro que possa definir um caso de uso para ele. Trabalhe em parceria com o líder GTM SAP da EMC em sua região a fim de obter ajuda para encontrar parceiros.

  2. Garantir alta disponibilidade e proteção dos dados

    O SAP HANA está amadurecendo. Os clientes estão procurando parceiros que possam fornecer orientação e soluções para proteger dados, garantir alta disponibilidade e reduzir o impacto sobre RTO (Recovery Point Objective, objetivo de ponto de recuperação) e RPO (Recovery Time Objective, objetivo de tempo de recuperação) no caso de tempo de inatividade planejado ou não planejado.

    A EMC implementou o HANA e pode compartilhar práticas recomendadas. Apresente a divisão BRS (Backup Recovery Systems) da EMC, a tolerância a desastres, a recuperação de desastres estática e os aspectos positivos característicos da EMC para clientes SAP.

  3. O appliance HANA pré-integrado é rígido e caro

    A estratégia de implementação inicial da SAP para o HANA, um appliance pré-integrado certificado, é comprovadamente um desafio para os clientes devido a seu custo e ao rígido BOM (Bill of Materials). Como alternativa, muitos clientes preferem aproveitar o armazenamento existente.

    Informe ao cliente que a SAP oferece o programa TDI (Tailored Datacenter Integration) para aproveitar o armazenamento existente. O cliente deve contatar um representante SAP para obter a indicação.

Valor da EMC

  • Appliance scale-out pré-integrado da Cisco ou da VCE

    "A EMC firmou parceria com a Cisco e a VCE para oferecer uma família de appliances certificados em SAP, baseados nos servidores blade Cisco UCS e no armazenamento VNX 5300. O appliance "Corporativo" apresenta dimensionamento de 16 nós e o appliance "Inicial" conta com dimensionamento de 4 nós, cobrindo o intervalo de SMBs e PoCs até as implementações de classe empresarial.

    A API de block da SAP, desenvolvida em parceria com a EMC, proporciona integridade de dados e failover sem interrupções. A plataforma Cisco UCS apresenta alta disponibilidade, gerenciamento completo e simplificado e correção automatizada de servidor, armazenamento e dispositivos de rede.

  • Proteção de dados e tolerância a desastres para SAP HANA

    Backup e recuperação: O Data Domain está disponível para backup NFS do HANA e pode ser gerenciado a partir do HANA Studio. A SAP certifica software para backup fornecido por terceiros e integrado ao BR Tools para HANA. O DD Boost é agendado para certificação do 1º trim.. Atualmente, os clientes podem usar o Networker caso desenvolvam um script personalizado.

    Tolerância a desastres: A SAP certifica a proteção síncrona DT (Disaster Tolerance, tolerância a desastres). A EMC oferece uma solução certificada pela SAP, com proteção síncrona fornecida pela MirrorView/S.

    Recuperação de desastres: A SAP não publicou uma estratégia de proteção assíncrona para o HANA, mas o Data Domain pode ser usado para replicação de longa distância e recuperação de desastres estática.

  • Integração personalizada de datacenter (TDI) para HANA

    Para reduzir os custos com hardware e operações, o SAP HANA agora pode ser usado com a plataforma de armazenamento preferida pelo cliente. Isso permite que a EMC apresente o VMAX quando o cliente não quiser usar o modelo de appliance.

    O DI (Digital Intermediate, intermediário digital) foi lançado no final de 2013, e os VMAX 10K, 20K e 40K são plataformas de armazenamento certificadas para TDI. Envolva os clientes interessados logo no início para aproveitar e preservar o espaço ocupado de armazenamento EMC.

    Os candidatos ideais são: 1) Clientes EMC VMAX que querem aproveitar o armazenamento existente e 2) Clientes comprometidos com servidores Cisco, VCE ou IBM para HANA.

A concorrência

O que a concorrência diz

A IBM usa a linha de negócios para definir o caso de uso HANA com SAP RDS (Rapid Deployment Solutions, soluções de rápida implementação), o que gera resultados rápidos que comprovam o valor do HANA, um pré-requisito para o investimento.

A IBM oferece dois tipos de appliances: uma solução de único nó para implementações menores e uma solução de scale-out que aproveita o GPFS (General Parallel File System, sistema geral de arquivos paralelo). O GPFS aproveita os discos locais e não exige um array externo.

Para DT (Disaster Tolerance, tolerância a desastres), a IBM está apresentando um GPFS certificado pela SAP.

Resposta da EMC

O HANA é um debate estratégico. Envolva proprietários de aplicativos e a TI para identificar e validar um caso de uso. Utilize um SI ou parceiro local que trabalhe com o cliente.

O GPFS adiciona um alto nível de complexidade ao ambiente HANA e exige um conjunto de habilidades adicionais para o gerenciamento.

Se a resposta ao modelo do appliance for NÃO e o cliente NÃO quiser um servidor Cisco, você poderá apresentar o TDI* como opção para proteger o armazenamento. Considere as implicações de uma parceria com a Cisco. Apresente o Data Domain para BRS. *O cliente deve solicitar ao representante SAP o suporte para TDI

O que a concorrência diz

A HP apresenta o SAP RDS (Rapid Deployment Solutions, soluções de rápida implementação), o que permite que a linha de negócio gere resultados rápidos que comprovam o valor do HANA, um pré-requisito para o investimento. A HP oferece três configurações de appliances: uma configuração de único nó para um volume de dados menor com armazenamento interno e duas configurações de scale-out. Uma configuração baseada em EOL EVA e IBRIX ainda está sendo oferecida, assim como um novo modelo baseado no armazenamento 3PAR. A HP é certificada pela SAP para DT (Disaster Tolerance, tolerância a desastres).

Resposta da EMC

Envolva proprietários de aplicativos e a TI. Identifique um caso de uso e apresente um resultado rápido. Trabalhe junto com um SI ou parceiro local para ampliar o debate e realizar uma POC (Proof of Concept, prova de conceito). Tenha cautela com a solução EVA/IBRIX oferecida pela HP. Ela é mais econômica que o 3PAR, mas gera impacto no desempenho. A solução 3PAR foi oferecida para elevar os recursos de desempenho da HP ao nível dos recursos da EMC, mas seu preço é alto. Se a resposta ao modelo do appliance for NÃO e o cliente NÃO quiser um servidor Cisco, você poderá apresentar o TDI* como opção para proteger o armazenamento. Considere as implicações de uma parceria com a Cisco. Apresente o Data Domain para BRS. *O cliente deve solicitar ao representante SAP o suporte para TDI

O que a concorrência diz

O appliance scale-out Cisco HANA com NetApp tem configuração igual à da EMC, exceto pelo back-end de armazenamento, que é NetApp. A solução NetApp aproveita o NFS do armazenamento compartilhado.

Como o HANA está na memória, o armazenamento é principalmente para persistência dos dados SAP. A abordagem da concorrência foca o componente de servidor, neste caso Cisco, e o modo como ele supera os servidores de outros fornecedores.

A NetApp pretenderá influenciar a Cisco para acessar contas da EMC.

Resposta da EMC

A EMC não deve se esquecer de que a NetApp vai tentar penetrar nas oportunidades da Cisco/EMC. É importante fortalecer o relacionamento e o planejamento de conta com seu interlocutor Cisco.

Observe a natureza legada do NFS como um sistema de arquivos compartilhado e os problemas técnicos conhecidos do NFS e da disponibilidade. A EMC desenvolveu a API de block para SAP em parceria com a SAP, para onde a maioria dos fornecedores está mudando.

Se a EMC precisar mudar de estratégia e propuser o TDI (Tailored Data Center), considere o impacto, porque a Cisco pode apresentar à NetApp.

O que a concorrência diz

A Fujitsu se concentra em equipes de TI que planejam, constroem e gerenciam serviços para HANA.

A Fujitsu oferece duas configurações de appliance: a configuração de nó único, para pequenos volumes de dados com armazenamento interno para persistência, e uma configuração de scale-out com armazenamento unificado NetApp.

A Fujitsu apresenta a DT (Disaster Tolerance, tolerância a desastres) para SAP, além de backup e recuperação integrados.

Resposta da EMC

Para o Appliance HANA, a VCE e a Cisco oferecem appliances para competir com a Fujitsu.

Observe a natureza legada do NFS como um sistema de arquivos compartilhado. A EMC desenvolveu a API de block para SAP em parceria com a SAP a fim de acabar com os problemas associados a NFS e HANA.

Para a EMC, é importante focar os proprietários de aplicativos e a TI, ajudando-os a identificar um caso de uso que apresente um resultado rápido que comprove o valor do HANA. Trabalhe junto com o SI ou parceiro local para ampliar o debate e obter resultados rápidos.

Perguntas de qualificação

  • TI: Você está pensando em implementar o SAP HANA? Você desenvolveu uma estratégia de consumo e um plano para lidar com alta disponibilidade, proteção de dados e recuperação de desastres?

    Como na arquitetura HANA o armazenamento é usado para persistência dos dados, vamos nos distinguir da concorrência principalmente com a disponibilidade e a proteção dos dados. As empresas que querem adotar o HANA tendem a procurar parceiros que possam ajudá-las a cuidar da proteção dos dados, da alta disponibilidade e da continuidade de negócios nessa plataforma em memória. Aproveite essa oportunidade de falar sobre a história e os principais aspectos positivos da EMC.

  • TI do aplicativo: Você está pensando em implementar o SAP HANA para melhorar o sistema de análise de informações em tempo real do SAP e aumentar a visibilidade dos dados SAP? Você definiu um caso de uso para o SAP HANA?

    A despesa e a quebra do paradigma com o HANA exigem que os clientes desenvolvam um caso de uso para comprovar o valor antes da adoção. Associe-se a um SI ou parceiro local que possa gerar um caso de uso com o cliente. Trabalhe com a Cisco ou a VCE para que o appliance passe por uma POC (Proof of Concept, prova de conceito).

  • Gerenciamento sênior de TI: Sua empresa está pensando em adotar o HANA como plataforma estratégica para SAP? Você tem interesse em conhecer as abordagens que comprovam que o HANA é “pronto para datacenters” para proteger sistemas e dados?

    Lembramos que como na arquitetura HANA o armazenamento é usado para persistência dos dados, vamos nos distinguir da concorrência principalmente com a disponibilidade e a proteção dos dados. Os executivos valorizam as parcerias com o HANA que fornecem ofertas de nível empresarial. Mostre os casos de uso planejados para HANA: data mart, BW e/ou pacote corporativo. Entenda com quem eles estão trabalhando.

    Criar valor: informe que a EMC usa o HANA, tem soluções comprovadas de nível empresarial, e pode compartilhar as experiências e práticas recomendadas da própria EMC.

Lidando com objeções:

  • O SAP HANA funciona em memória, usando armazenamento apenas para persistência. Qual é o valor fornecido pelo armazenamento?

    Lembre aos clientes que o SAP é essencial para as empresas. Se os servidores caem, os dados em memória são perdidos. Agora a SAP está disponibilizando o HANA para ERP e Business Suite. As empresas precisam e devem esperar os mesmos níveis de alta disponibilidade, proteção de dados e continuidade dos negócios com o HANA disponíveis nos modelos DBMS tradicionais.

    A EMC trabalhou em parceria com a SAP nas soluções e práticas recomendadas para proteção de dados e ambientes SAP para garantir os melhores SLAs para RPO e RTO. O HANA dá continuidade a essa proposta.

  • Somos clientes da EMC e queremos aproveitar o VMAX que já usamos, mas o VMAX não é certificado com um appliance de fornecedor de servidor.

    Agora a SAP oferece a oportunidade de executar o servidor SAP HANA com o armazenamento preferido pelo cliente para reduzir o custo das operações e de hardware nos clientes da base instalada com o aproveitamento dos processos de operação e componentes de hardware existentes. A opção da SAP é o TDI (Tailored Datacenter Integration).

    No momento, a SAP está realizando vários projetos piloto com clientes para o SAP HANA TDI. Os clientes podem ser nomeados por seus respectivos representantes de contas SAP do programa.
