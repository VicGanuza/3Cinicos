Otimização de aplicativo preexistente

Backup e recuperação para SAP

  • Os bancos de dados SAP com muitos TBs obrigam muitos clientes a clonar o armazenamento tier-1, descarregá-lo em fita e gerenciar restaurações incrementais. Esse ambiente é ideal para o EMC BRS de última geração, com desduplicação.

Três principais desafios

  1. Pressão sobre as janelas de backup do SAP

    Os bancos de dados SAP com muitos TBs obrigam muitas empresas a clonar o armazenamento tier-1, de custo alto, para acelerar o backup. Essa estratégia de clonagem é arriscada, já que a ausência de backups completos requer a coordenação de restaurações incrementais. Além disso, o processo de localizar e devolver ao local as fitas armazenadas externamente é demorado.

    O Data Domain foi um grande achado. A Callaway Golf alcança taxas de desduplicação de 25:1 para SAP, com desempenho de mais de 1 TB por hora, permitindo mais backups em disco completos para facilitar a restauração.

  2. Custo: o espaço físico do SAP inclui grandes sistemas de não produção

    Os ambientes SAP que não são de produção, inclusive DR, suporte a produção, QAS, testes, desenvolvimento e treinamento, consomem mais armazenamento do que os de produção, mas também precisam ser protegidos. Se a produção ocupa 3 TB, um único backup pode chegar a 12 TB (de três a quatro vezes o tamanho da produção).

    Os benefícios do backup desduplicado se estendem aos ambientes que não são de produção, inclusive aos backups completos de todo o ambiente SAP, bem como à recuperação baseada em disco, que é mais rápida.

  3. Administradores SAP e DBAs precisam de visibilidade e controle do armazenamento

    Para Oracle: a SAP tem ferramentas para gerenciar backup e recuperação. Três produtos de software são certificados com SAP BRTools para Oracle, o que permite que os administradores SAP gerenciem backups integrados com software para backup centralizado. Soluções de backup fornecidas por terceiros

    O NetWorker pode ser integrado a APIs de fornecedores específicos, inclusive com o SAP BRtools for Oracle Database, eliminando a necessidade de desenvolvimento e manutenção de scripts personalizados.

Valor da EMC

  • A desduplicação do EMC Data Domain é uma prática recomendada para backup e recuperação SAP.

    Bancos de dados de produção SAP com muitos TBs são comuns e crescem exponencialmente quando você inclui ambientes que não são de produção (teste, desenvolvimento, QAS e DR). O resultado em termos de custo e complexidade é: a) o armazenamento tier-1 e a clonagem podem ser usados para administrar as janelas de backup; b) backups incrementais geram janelas de recuperação maiores e complexas; e c) ambientes que não são de produção triplicam ou quadruplicam o volume de dados de backup.

    Comprovação: a Callaway Golf atinge taxas de desduplicação de 25:1, com desempenho de mais de 1 TB/hora em produção, permitindo mais backups completos. Os custos de gerenciamento de um ambiente sem fitas agora foram eliminados.

  • Gerenciamento unificado e administração para SAP Basis por DBAs e administradores de backup

    A integração do NetWorker com banco de dados SAP e SAP BRtools for Oracle oferece total visibilidade e gerenciamento unificado para DBAs e administradores SAP, eliminando a necessidade de desenvolver e manter scripts personalizados.

    Comprovação: a Sports Chalet usa a desduplicação do EMC Avamar com o EMC NetWorker para unificar o gerenciamento de backup e recuperação. Com o Avamar, os backups SAP são concluídos em 20 minutos, em vez de 16 horas. Com o NetWorker, os administradores dedicam apenas 5 a 10 minutos ao gerenciamento de backup, em vez das horas antes necessárias.

    OBS.: para informações sobre o NetWorker com o SAP HANA, consulte a execução HANA.

  • Como facilitar a recuperação de desastres para SAP

    Alguns clientes SAP preferem uma solução de backup e recuperação de custo mais baixo para distâncias assíncronas que adote uma estratégia de reinício estático. O Data Domain permite que os clientes SAP ampliem os backups locais, propiciando uma recuperação de desastres dimensionável e eficiente com tecnologias de replicação que reduzam os requisitos de largura de banda em até 99% quando os dados são movimentados para locais distantes.

    Comprovação: a Callaway Golf está usando o backup desduplicado do Data Domain para facilitar a recuperação de desastres, replicando os dados de backup de San Diego em Carlsbad, Califórnia. O custo é reduzido devido à reutilização da largura de banda WAN em outros processos de negócios essenciais.

A concorrência

O que a concorrência diz

O Tivoli Storage Manager (TSM) é oferecido pela IBM para gerenciamento de dados SAP, inclusive backup e recuperação. O TSM é certificado com o SAP BRtools para Oracle, MAS NÃO com o SAP HANA.

Resposta da EMC

O EMC NetWorker, que também é compatível com DB2 e Oracle DBMS para SAP, é certificado com o SAP BRtools para Oracle. Aproveite os recursos técnicos da EMC para fornecer diferenciação profunda em relação à concorrência e referências de sucesso.

O que a concorrência diz

O Symantec NetBackup é certificado com os bancos de dados SAP BRtools para Oracle e SAP HANA. Talvez a Symantec tente aproveitar o certificado HANA, o que não deve ser relevante para muitos usuários SAP hoje.

Resposta da EMC

O EMC NetWorker é certificado com o SAP BRtools para Oracle, MAS NÃO com o SAP HANA. Aproveite os recursos técnicos da EMC para fornecer diferenciação profunda em relação à concorrência e referências de sucesso.

Perguntas de qualificação

  • TI: Como você está reduzindo o custo dos backups SAP? Você tem flexibilidade para realizar backups completos diariamente?

    Geralmente os clientes criam clones para armazenamento tier-1 com snapshots incrementais. Os backups completos podem ficar restritos apenas aos fins de semana. Fitas são usadas a longo prazo. Discuta abordagens alternativas com a oportunidade de backups completos diários.

    Inicie o debate sobre a desduplicação e os métodos para o cliente aumentar o número de backups completos, possivelmente com o armazenamento diário em um appliance de backup integrado de baixo custo. Comprovação: mencione Callaway Golf, Sports Chalet, Aramis e outros clientes SAP.

  • TI: Você executa o SAP em Oracle e usa o SAP BRtools?

    Se o cliente executar o SAP em Oracle, a integração com o BRtools exigida para gerenciar o backup para SAP baseado em Oracle.

    Explique que o NetWorker pode ser integrado com APIs de fornecedores específicos, inclusive com o SAP BRtools for Oracle, eliminando a necessidade de desenvolvimento e manutenção de scripts personalizados.

  • TI: Qual é sua estratégia para SAP? O que você usa hoje?

    A recuperação de desastres é importante para o cliente, embora geralmente não seja bem implementada. Procure clientes SAP que usam fitas como estratégia de recuperação de desastres. Alguns clientes SAP não têm um plano implantado devido ao custo, principalmente em razão de distâncias assíncronas.

    Comprovação: mencione a implementação do Data Domain pela Callaway Golf.

Lidando com objeções:

  • Os bancos de dados SAP têm altas taxas de mudança e registros de arquivo/transação que não os tornam candidatos fortes para a desduplicação.

    Muitos métodos de desduplicação não são suficientes nesse ambiente. Mas a desduplicação com comprimento variável do Data Domain alcança taxas de desduplicação líderes do setor no espaço de backup do banco de dados. Os clientes com taxas de mudança diárias de 10 a 15% e retenção de 4 a 6 semanas estão alcançando taxas de desduplicação entre 7 e 10 vezes mais altas.

    A Callaway Golf, um cliente SAP com banco de dados de muitos TBs, está atingindo taxas de desduplicação de 25:1 nos locais principal e de recuperação de desastres da empresa.

  • Meus administradores SAP e DBAs querem ter visibilidade direta: não querem depender de administradores de backup.

    O NetWorker permite que os administradores SAP e DBAs tenham visibilidade e controle direto. O NetWorker também pode ser integrado com APIs de fornecedores específicos, como o SAP BRtools for Oracle Database, eliminando a necessidade de scripts personalizados.

    No caso do Data Domain, o utilitário do banco de dados nativo (Oracle RMAN, dumps do SQL, etc.) permite que os DBAs mantenham o controle. Com isso, os DBAs podem gravar em um sistema Data Domain que funciona como plataforma consolidada de armazenamento e proteção para toda a empresa.
