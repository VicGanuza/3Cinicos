サービスとしてのITの運用

ホスト型プライベート クラウド（サービス プロバイダー）

  • お客様は、信頼できるEMCサービス プロバイダー パートナーを利用して、プライベート/ホスト型プライベート/パブリックの各クラウドで特定のワークロードを供給します。適切なパートナーの特定と商談の進展をお手伝いします。

上位3つの課題

  1. サービスを向上させるためのIT予算が限られている

    IT部門は、より少ないリソースでより多くを達成することが求められています。ITサービスの向上と高いパフォーマンスが期待されています。企業は、ITの設備投資と運用コストを削減し、運転資本利益率を短期間で向上させる必要があります。このような課題から、お客様は、どのワークロードが社外展開に最も適しているかを評価するためのサポートを必要としています。

    EMCサービス プロバイダー パートナーは、ワークロードを最適化するための「サービスとして」オプションの提供を通じて、コスト削減、効率の向上、アプリケーションの可用性の向上を支援できます。

  2. 現在のIT環境内は俊敏性に限界がある

    IT部門は、ビジネスに対する革新的なソリューションをオンデマンドで提供できることがますます求められています。静的なアーキテクチャと旧式のプロセスでは、複雑さとコストを嫌って、変更を避けることが珍しくありません。ワークロードやアプリケーションをオンサイトに残した方が、リスク回避が容易だという認識が存在します。

    EMCサービス プロバイダー パートナーは、多様な「サービスとして」オプションを提供しています。IT部門は、プライベート クラウド、ホスト型プライベート クラウド、パブリック クラウドのソリューションを利用して、ワークロードを導入し、タイム トゥ バリューを短縮できます。

  3. 制御とセキュリティが失われている

    エンタープライズ レベルのアプリケーションを社外に移行する際の主要な懸念事項は、セキュリティに関するものです。リスクを軽減するために、企業は、規制要件やパフォーマンス要件を根拠として、ワークロードの移動を制限しています。その結果、社外モデルであればメリットが得られるようなワークロードが、ITインフラストラクチャ上に残ることになります。

    EMCサービス プロバイダー パートナーは、ワークロードの安全な移行の遂行を専門としています。エンタープライズ レベルの信頼できる専門技術によって、クラウドへのシームレスな移行を保証します。

EMCの価値

  • EMCサービス プロバイダー パートナーが効率の向上を通じて、ITとLoB（基幹業務）アプリケーションを変革します。

    EMCはサービス プロバイダー パートナーと連携して、お客様がサービスをオンデマンドで提供し、ITのチャージ バックに対する可視性を高めるための支援を提供します。アプリケーションを社外に移行することで、SLAとサービスは向上し、ITのリソースと支出は合理化されます。IT資産とインフラストラクチャに対する設備投資を削減し、これらの支出を運用コストの予算に移すことができれば、効率向上が実現します。ITの生産性とコスト削減が向上することで、ビジネスへのレスポンス タイムとサービス提供が短縮されます。リソースの最適化が、EMCサービス プロバイダー パートナーが提供する「従量課金制」と「サービスとして」オプションを通じて達成されます。

  • EMCサービス プロバイダー パートナーが備える俊敏性と柔軟性

    EMCはサービス プロバイダー パートナーと連携して、競争優位性を実現するための俊敏性、規模、将来にわたって利用可能なITに関するコンサルティングをお客様に提供します。アプリケーションとワークロードを、EMCのテクノロジを基盤としたEMC Powered Serviceを使用するサービス プロバイダーにオフロードできます。お客様は、アプリケーションの展開、開発、移行を実行する方法を、プライベート クラウド モデル、ホスト型プライベート クラウド モデル、パブリック クラウド モデルから選択できます。EMCは、EMCサービス プロバイダー パートナーをプロモートし、その技術チーム、セールス チーム、マーケティング チームを支援し、成功に貢献し、競合を避けています。最適なワークロードの配置と機能を特定するためのツールも提供しています。

  • EMCサービス プロバイダー パートナーが備える信頼できるセキュリティと専門技術

    EMCが備える検証済みテクノロジ スタックのポートフォリオと、サービス プロバイダーが備える専門技術は、強力な組み合わせです。アプリケーションとワークロードを移行する際、お客様には、プライベート クラウド、ホスト型プライベート クラウド、パブリック クラウドのいずれであるかにかかわらず、移行と接続の簡略化が必要になります。今日の環境では、コンプライアンス、セキュリティ、信頼が最優先事項です。EMCとEMCサービス プロバイダー パートナーは、コンサルティング型の評価を提供し、ワークロードの要件を満たすセキュリティ、可用性、ビジネス継続性の提供を確保します。

競合製品

競合他社の位置づけ

AWSは、パブリック クラウドの「先発者」として、サービスとしてのインフラストラクチャ市場を独占しています。技術革新、包括的なサービス、豊富なAPI、強力な開発サポートも、サービスとしてのプラットフォームにおける同社のリードを後押ししています。AWSには、長期契約のない、競争力のある従量課金制の料金が設定されています。クレジット カード/ギフト カードによるサービス アクティベーションで簡単にサービスを開始でき、人とのやり取りが必要ありません。

EMCの対応

EMC Powered Servicesは、プライベート クラウド モデル、ホスト型プライベート クラウド モデル、パブリック クラウド モデルをサポートし、AWSでは利用できない制御と選択肢をエンタープライズITに提供します。EMCグローバル サービス、EMCサービス プロバイダー パートナー、リセラーそれぞれのITプロフェッショナルとEMCの強力なパートナー関係は、意義のあるSLA、セキュリティ、監査とコンプライアンス、堅牢なサポートの重要性に対する理解が証明された上に構築されています。

競合他社の位置づけ

IaaS市場に参入したばかりのGoogle Cloud Platformは、検索業界の独占をしながらエンタープライズ品質のメールとアプリケーションのサービスを提供することで得た豊富な経験を足がかりとしています。Google App Engine、Google Compute Engine、Google Cloud Storage、Google BigQueryは、AWSにおける同等のサービスと競合していますが、企業の採用例はさほど多くはありません。Googleは価格設定が積極的で、クレジット カードを使用して簡単にサービスを開始できます。

EMCの対応

信頼と経験が、EMC Powered ServicesとGoogleのクラウド製品との差別化要因です。EMCは、データセンターからクラウドにワークロードを移植する際に、実証済みのセキュリティ、高可用性、業界をリードするアプリケーション パフォーマンスを提供しています。EMCと連携することで、ITプロフェッショナルは、有効期限の限られたホスト型サービスや頻繁に変更されるAPIに頭を悩ませることがなくなります。EMCがパフォーマンスと安定性を提供します。

競合他社の位置づけ

Microsoftは、エンタープライズITをターゲットに、IaaS、PaaS、SaaS向けのパブリック クラウド プラットフォームであるWindows Azureを提供しています。MSは、エンタープライズをExchangeからOffice 365に転換したいと考えており、データに対するクラウドのようなアクセスを提供することで、ライセンス コストとPCメンテナンス コストの削減を実現しています。Azureは、積極的なタイム トゥ マーケット、柔軟性、最小制限のない分単位の価格設定が特徴です。PaaSとしては、クラウドの限定版MSDNライセンスをサポートしています。

EMCの対応

柔軟性、選択肢、ネームド アカウントが、EMCのサポートするクラウドのAzureに対する強みです。Hyper-Vに対しても、EMCは、Linuxなどの、Microsoft以外の仮想化とオペレーティング システム環境をサポートしています。Microsoftのクラウドには、Xbox 360やBingのような「専属の」お客様は多くありません。EMCとEMCサービス プロバイダーおよびリセラー パートナーは、Fortune 500社の99%以上、世界各地の何千社もの企業をサポートしています。

見極めのための質問

  • 経営陣レベル、VP、事業部、IT： 今は実行できていないが将来は実現したいことが何かありますか。ビジネスやITのイニシアティブを変革し、変化に影響を与え、競争力を向上させるための計画がありますか。

    この質問によって、ITの変革を通じた戦略的ビジネス目標の達成に関するディスカッションを主導します。経営幹部の言葉で話し、 市場シェアの獲得、新しいビジネス イニシアティブ、ITの活用による収益拡大などに触れます。これを通じて、EMCとEMCサービス プロバイダー パートナーは、適切なクラウドモデルを選択することで競争優位性を実現できるビジネス インパクトについて伝えるチャンスが得られます。企業のビジョン、方向性、イニシアティブを理解することに重点を置いてください。

  • CFO、CIO、IT： XaaS戦略はお持ちですか。アプリケーションをクラウドに移行したいとお考えですか。現在、サービス プロバイダーを使用してワークロードをクラウドにオフロードされていますか。

    お客様がすでにクラウド オプションを活用しているかどうかを見極めます。お客様がEMCサービス プロバイダー パートナーからサービス提供を受けている場合は、既存の信頼関係を基盤として、クラウド インフラストラクチャの可能性について会話をするチャンスがあります。お客様が「サービスとして」サービスの市場を調査している場合は、EMCのサービス プロバイダー エコシステムの価値を伝えます。EMCとサービス プロバイダーの主な差別化要因として、 機能、専門技術、セキュリティ、柔軟性について説明します。

  • 経営陣レベル、VP、事業部、IT： どのクラウド オプションが御社のサービス提供のニーズに最適ですか。どのワークロードをどこに配置する必要がありますか。現在はどのようにしてサービスの可用性を確保していますか。

    組織がクラウド サービスのメリットを真の意味で理解しているかどうかを見極めます。「1つのサイズですべてをまかなう」というアプローチは、パフォーマンスと可用性に対して複雑なニーズを持つ企業には適しません。EMCはサービス プロバイダーと連携して、ストレージ、バックアップ、分析、アプリケーション、セキュリティに関する幅広いソリューションを提供しているという事実をアピールします。サービス プロバイダーとEMCテクノロジの組み合わせは、ホスト型プライベート クラウドにおいて高い可用性とワークロード パフォーマンスを確保するソリューションを提供しています。

オブジェクション・ハンドリング

  • 「パブリック クラウドだと、ハッカーによる攻撃などのセキュリティ問題に対してデータが脆弱になりますね。」

    EMCなら、プライベート クラウド、ホスト型プライベート クラウド、パブリック クラウドのどのモデルにおいても、信頼性と制御を確保できます。EMCサービス プロバイダー パートナーは、オンプレミスITの場合と遜色ないセキュリティを提供できます。EMCとEMCパートナーが提供する最先端のテクノロジ ソリューションによって、ワークロードがホスト型プライベート クラウドに移行された場合のリスクが排除されます。経験の乏しいクラウド プロバイダーだと、エンタープライズ クラスのアプリケーションの管理に関する業界経験がなく、ソリューションが規制要件を満たしていないことも少なくありません。

  • 「サービスをクラウドに移行すると、IT部門が制御できなくなります。それはスタッフに悪影響を与えます。」

    EMCサービス プロバイダーは、基幹業務とIT部門の両方にメリットをもたらすソリューションを提供しています。ITが優れたサービスを提供すれば、主要なビジネス パートナーとしてのイメージが強化されます。運用タスクとワークロードを移行し、優先順位づけを行うことで、リソースを革新的かつ戦略的なプロジェクトに割り当て直すことができます。リソースを最適化できれば、コストは削減され、効率は向上します。ビジネス目標の達成に向けたIT部門やITプロフェッショナルの専門技術の活用を、これまで以上に進めることができます。

  • 「AmazonやGoogleならコストがもっと低く済むし、『as-a-service』市場でもリーダー的地位にあります。」

    「AmazonやGoogleでは、あらかじめパッケージ化されたソリューションを、クレジット カードで手軽に購入できますが、長期的な実質コストの透明性には疑問があります。また、セキュリティ、パフォーマンス、収益性の問題も懸念事項です。それに代わるものとして、EMCテクノロジとサービス プロバイダーの専門技術の組み合わせは、エンタープライズ レベルのソリューションを提供します。コモディティ化したパブリック クラウド プロバイダーの場合、プライベート クラウド、ホスト型プライベート クラウド、パブリック クラウドを提供するEMCサービス プロバイダーとは異なり、マルチ テナント サービスに限界があります。」
