实现 IT 即服务

EMC VSPEX 经验证的基础架构

  • VSPEX 参考体系结构经过 EMC 验证，并由 EMC 渠道合作伙伴独家打包和销售
  • 要确定支持 VSPEX 的渠道合作伙伴，请与当地 PDM 联系

三大挑战

  1. 传统数据中心： IT 预算困境

    公司在采用传统 IT 模式时，想要在不增加复杂性和大量成本的情况下扩展他们的环境简直就是天方夜谭。这是因为硬件与它支持的应用程序紧密相关。随着时间的推移，这种模式已经到了穷途末路，因为所有资金只是用于维持正常运转，而不是创新。

    VSPEX 可以将资源从专用于单个应用程序的硬件思洛转移到高效、简单且灵活的虚拟化资源池。

  2. 大规模数据增长

    所有企业都在应对爆炸式数据增长。由于通过网络发送的数据量非常巨大，因此显著降低了应用程序的性能。据预计，数字世界将增大到 40 ZB，对于还没有做好准备的组织而言，问题只能会变得更糟。

    VSPEX 可以虚拟化客户的数据中心，以便他们能够以最高效的方式利用其基础架构，从而出色地解决如今面临的大规模数据增长难题。

  3. 我知道有问题，但我缺少专业知识和时间

    公司通常已经意识到在他们的数据中心内存在的问题，并且知道当前的运营模式不具有可持续性。但是，这些公司在内部没有专业知识、时间和资源来实施虚拟化私有云模式。

    使用 VSPEX 时，解决方案的设计、配置、测试和验证全部都由 EMC 来做。本解决方案由我们的专业渠道合作伙伴提供。如此一来，便减少了部署云所需的专业知识、时间和资源。

EMC 价值

  • 简单

    VSPEX 经过 EMC 的严格测试和验证。VSPEX 旨在从部署私有云中消除复杂性，简化部署过程。VSPEX 让客户可以灵活地选择他们在虚拟机管理程序、服务器和网络层方面所需的技术品牌，并将其与 EMC 业界领先的存储、备份和数据保护相结合。VSPEX 的部署、安装和配置全部都在参考体系结构中获取，这意味着客户可以简单、快速地部署 VSPEX，从而缩短价值实现时间并最大程度地提高 ROI。

  • 高效

    VSPEX 可以提高 IT 层每个级别的效率。诸如精简资源调配、重复数据消除和数据压缩等存储效率技术可以将前期成本降低多达 50%。EMC 闪存优先策略可以动态吸收系统工作负载的意外高峰。VSPEX 的体系结构可以充分发挥系统中所有 VNX 的 Intel 多核心 (MCx) 的强大威力，该系统经过优化，将所有服务（RAID、IO、FAST Cache、DP）以统一方式平均分配在这些核心上，从而将性能提高多达 4 倍。VSPEX 与 EMC 备份结合使用可以将备份时间缩短 90%，将恢复速度提高 30%。

  • 灵活

    VSPEX 可以提供更高的灵活性，客户可以选择他们偏爱的虚拟机管理程序、服务器和网络技术，并将其与 EMC 业界领先的存储、备份和数据保护技术结合使用。由于 VSPEX 经过 EMC 测试和验证，因此客户可以快速部署 VSPEX，并对整套产品的互操作性和性能充满信心。

竞争

竞争对手推介

FlexPod 与 NetApp FAS 统一存储体系结构、Cisco UCS 和 Cisco Nexus 交换机相结合，构成可扩展参考体系结构，主要面向中端市场及大型企业组织和 SP。FlexPod 针对大量工作负载进行了验证，包括 VMware vSphere、Microsoft 私有云和应用程序、SAP、使用 Citrix XenDesktop 和 VMware View 的 VDI、Red Hat 及其他。

EMC 的回应

FlexPod 没有提供实际的灵活性。客户必须使用 NetApp 存储和 Cisco UCS 服务器及 Cisco Nexus 交换机。使用 VSPEX，客户可以选择他们所需的服务器和网络技术，包括 Cisco UCS 和 Cisco Nexus 技术。事实上，我们拥有大量适用于 VSPEX 的 Cisco 验证设计，这些设计详细说明了使用 Cisco UCS 和 Cisco Nexus 解决方案测试和验证的 VSPEX。

竞争对手推介

Dell 的活动基础架构产品组合包括预配置的 RA 解决方案，将服务器、存储、网络、虚拟化和统一管理集成在一个可扩展解决方案中。Dell 提供单一供应商支持模式，从而消除了耗时的帮助请求过程。活动基础架构包括两个主要的解决方案系列： Dell vStart 和 Dell Active System 800。

EMC 的回应

Dell 提供全套硬件，这表示 Dell 限制了客户的选择。使用 VSPEX，客户可以选择他们偏爱的服务器、网络和虚拟机管理程序品牌，并与 EMC 业界领先的存储、备份和数据保护技术完全集成在一起。

竞争对手推介

HP VirtualSystem 是预配置的 RA 解决方案产品组合，适用于小型到大型企业和服务提供商部署。HP VirtualSystem 解决方案以 HP ProLiant 或 HP BladeSystem 服务器、HP StoreVirtual 或 HP StoreServ 存储和 HP 5900 系列或 HP Flex10 网络为基础而构建，可以消除虚拟化复杂性。

EMC 的回应

HP VirtualSystem 完全不能提供任何灵活性。在整个平台中使用 HP 技术，就等于锁定在一家供应商。VSPEX 允许客户在平台的虚拟机管理程序、服务器和网络层选择品牌。因此，如果客户需要，他们甚至可以在服务器层使用 HP 技术来部署 VSPEX。

竞争对手推介

Hitachi UCP Select 是一系列 RA 解决方案，提供包含服务器、存储和网络组件的应用程序优化解决方案。所有 UCP Select 解决方案中除 Hitachi UCP Select for vSphere with Cisco UCS（该解决方案将 Hitachi 虚拟存储平台与 Cisco UCS 刀片服务器和网络结合使用）外，都将 Hitachi 刀片和服务器与 Brocade 打包在一起。

EMC 的回应

尽管 HDS UCP Select 可以提供有限的灵活性，但不能像 VSPEX 提供的那么多。HDS UCP Select 要求客户要么选择将 Hitachi 存储与 Hitachi 服务器和 Brocade 交换机结合使用，要么选择将 Hitachi 存储与 Cisco UCS 和 Cisco 网络结合使用。使用 VSPEX，客户可以选择他们偏爱的服务器和网络品牌，而不是仅选择两种服务器和网络品牌。

竞争对手推介

PureFlex System 是预配置、预集成的基础架构解决方案，可以与基于 x86 或 POWER 处理器的计算节点配合使用，从而可以使用其他选项进一步进行自定义以满足客户精确的 IT 基础架构需求。PureFlex System 将计算、存储、网络、虚拟化和管理整合在一个基础架构系统中。

EMC 的回应

PureFlex 不够灵活。它完全锁定在一家供应商。IBM 提供解决方案中的所有硬件。如果客户想要在其解决方案中包含非 IBM 硬件，那么他们将无法获得 PureFlex。使用 VSPEX，客户可以选择将 IBM x86 服务器用作其 VSPEX 的一部分，这要归功于服务器层的灵活性。VSPEX 的优势在于可以提供真正的选择， 而 IBM 让客户别无选择。

资格确认问题

  • IT： 您目前如何处理不断增加的应用程序服务器要求？

    VSPEX 可以整合、虚拟化和简化所有应用程序服务器的管理，使客户能够摆脱物理服务器和直连存储上仅一个应用程序的境况。VSPEX 与新一代 VNX 相结合，可以提供统一存储，在一个平台上支持多个协议和多种磁盘类型（包括企业级闪存）。利用 VSPEX 中的 EMC 备份技术，客户可以将备份时间缩短 90%，恢复速度提高 30 倍，并将备份管理时间减少 81%。

  • IT： 能否给我描述一下如今您的企业在管理多个供应商的技术时遇到的挑战？

    EMC 对每个解决方案都进行了严格的测试和验证，以便在构建云解决方案时降低复杂性和风险，让客户能够针对 EMC 验证的系统自主选择供应商。VSPEX 通过利用 EMC 与配备 EMC Virtual Storage Integrator (VSI) 的 VMware 以及配备 EMC Storage Integrator (ESI) 的 Microsoft 产品的紧密集成，可支持在真正统一的存储平台上进行应用程序感知存储资源调配，从而使管理员可以通过虚拟机管理程序管理存储。

  • IT： 能否给我描述一下您认为在您的企业中部署虚拟化时遇到的障碍是什么？

    VSPEX 通过了 EMC 的严格测试，并且由 EMC 的专业渠道合作伙伴打包和交付。VSPEX 可以化繁为简，应对复杂的挑战。EMC 的渠道合作伙伴具备技术指南，有助于他们显著简化 VSPEX 的设计和部署以及在其中运行的应用程序，因此客户不必投入资源单独地解决问题。VSPEX 使企业愿意利用虚拟化来简化、优化和自动化 IT 环境。

异议处理

  • “我们过去在参考体系结构方面没有非常好的经验，我们需要灵活性。”

    VSPEX 将选择权交到您的手上。我们提供工具和调整，帮助您利用我们的专业知识自定义设计以满足您企业的确切需求，并且我们对经过充分测试和验证的解决方案充满信心。

  • “我们不希望放弃原有 IT 投资。”

    VSPEX 非常灵活。选择符合您的偏好并能服务于现有 IT 投资的虚拟机管理程序、服务器和网络品牌。VSPEX 可以与您已经购置的现有资源和软件许可证结合使用。

  • “我们当前的存储现在能够满足我们的需求。”

    您的存储就目前而言可能完全适用，但是随着数据呈爆炸式增长，从长远来看未必适用。基于 VNX 的 VSPEX 可以简化操作并节省资金，从而使您先行一步控制数据增长，走在虚拟化趋势的前列。
