存储即服务

ViPR 软件定义存储

  • 部署简单、可扩展的开放式软件存储平台
  • 将存储转变为全自动存储服务交付模型

三大挑战

  1. 复杂的异构存储环境

    大多数客户运行的是思洛式多供应商环境，在这种环境下，阵列需要单独管理并且工作负载与这些阵列紧密关联。

    部署单一软件定义存储平台可以简化、集中化和自动化存储管理和交付，从而降低成本、提高利用率，并动态满足现有基础架构中新使用情形和工作负载的要求。

  2. 现有基础架构不适应云

    开发人员绕过 IT 使用公共云存储。这不仅增加了成本和安全风险，而且 IT 丧失了对数据的控制。

    在现有的安全受控环境中快速部署私有云可以获得与公共云一样的优势和用户体验。

  3. 许多应用程序禁锢在单独的存储系统中

    现如今，使用不同的 API 构建新应用程序以满足不断变化的工作负载需求和要求会耗费大量时间。开发人员绕过 IT 使用公共云存储。这不仅增加了成本和安全风险，而且 IT 丧失了对数据的控制。

    通过提供独立于底层存储基础架构、跨阵列运行并可增加灵活性和选项的服务和应用程序，在 IT 组织中增加技术创新能力。寻找全新的云级使用工作负载和数据服务机会。

EMC 价值

  • 简化跨异构阵列的存储管理和交付

    ViPR 提供单一存储平台，可以跨异构存储集中、自动执行存储资源调配，简化管理，并最大程度地提高利用率。使用 ViPR，可以将控制平面与数据平面分离开。这样 ViPR 便可以将物理阵列中的存储抽象为单一存储池，以保留每个阵列的各个特征和值， 从而支持管理员定义基于策略的存储池，并通过自助式目录即时提供给用户。ViPR 可以集中管理存储资源、性能和利用率，包括计量和按存储容量使用计费。

  • 将合作范围扩展到 EMC 之外，支持使用第三方存储和云平台

    ViPR 提供使用标准 RESTful API 构建的轻量级、超大规模云体系结构，从而可以轻松地集成非 EMC 存储和云平台，并开发全球数据服务。目前，ViPR 支持与 NetApp、VMware、Microsoft 和 OpenStack 进行无缝集成。随着时间的推移，EMC 还将发布其他第三方平台集成。EMC 为客户、服务提供商和供应商提供丰富的 SDK 以编写存储适配器，这些适配器可以将任何存储平台（包括商用平台）集成到 ViPR 中。

  • 部署开放式平台，增加灵活性并提供更大的选择余地

    ViPR 数据平面支持通用 API，包括 Amazon S3、EMC Atmos 和 OpenStack。这使开发人员可以完全摆脱存储依赖关系编写应用程序和全球数据服务，从而动态地适应新使用情形和工作负载。现在，ViPR 提供对象数据服务以在基于文件的存储中以对象的形式存储、访问和操作非结构化数据（例如，图像、视频、音频、在线文档），而不必重新编写或改编应用程序。EMC 将继续通过开放 API 和 SDK 提供新的全球数据服务并培育开放式开发社区。

资格确认问题

  • 最高管理层级别高管： 公司是否要求您既要管理数据急速增长，又要减少运营支出？

    EMC ViPR 通过提供可以跨异构存储集中、自动执行存储资源调配、简化管理并最大程度地提高利用率的单一存储平台，提高 EMC 和第三方（例如 NetApp）存储的利用率。ViPR 还提供所有存储利用率的单一视图，以及按租户计量和按存储容量使用计费功能。管理员将这三种功能结合使用，可以摆脱重复性的手动任务，从而将精力集中在优化系统资源上。

  • 最高管理层级别高管： 公司是否要求您将 IT 从成本中心转变为利润中心，并为您的企业提供存储服务模式？按存储容量使用计费和实现 StaaS 重要吗？

    ViPR 支持您在现有基础架构中提供在成本方面与公共提供商相比持平或者更低的存储服务。组织使用 ViPR 通用 API（包括 Amazon S3 和 OpenStack）可以跨文件和数据块存储应用新的云级工作负载，而不必重新编写应用程序。ViPR 提供单一平台以按租户监视利用率，并提供涵盖 ViPR 管理的所有存储的计量和按存储容量使用计费信息。

  • IT： 较长的手动资源调配和存储交付周期是否影响业务敏捷性，并迫使开发人员绕开 IT 去寻找公共云服务？

    ViPR 通过支持用户自助访问存储，使组织收回对数据的控制权。ViPR 控制器可以将物理阵列中的存储抽象为保留每个底层阵列的所有值的单一存储池， 从而支持管理员定义基于策略的自动化存储池，并通过自助式目录即时提供给用户。

异议处理

  • “存储虚拟化解决方案已推出有一段时间了。EMC 过去就已经在这一领域做了尝试，为什么现在还在主推？它们有何不同之处？”

    EMC ViPR 转变了数据中心自动化、管理和呈现存储的方式。与将存储视为 JBOD 的虚拟应用装置不同，ViPR 控制平面（在这一平面管理存储）与数据平面（在这一平面存储数据）分离开。这种方法使 ViPR 将物理阵列中的存储抽象为单一存储池，以保留每个阵列的各个特征和值。

  • “为什么我需要使用 EMC 的另一个管理工具？我刚刚购买了 EMC Storage Resource Management Suite。”

    ViPR 提供单一管理工具来管理虚拟存储阵列和池利用率、计量和性能。ViPR 也可以将这些存储数据导入 SRM Suite 和 vCenter，以便跨 SDDC 进行端到端查看。

  • “是什么使 EMC 在存储虚拟化和软件定义存储领域与众不同？”

    ViPR 是世界上第一个真正的开放式、可扩展软件定义存储平台。与需要您购买其他硬件的竞争性产品不同，ViPR 在已有的硬件上运行，可以保持较低的成本开销，并提高利用率。ViPR 支持开放式访问北向和南向 API 以将 ViPR 扩展到第三方存储（包括商用存储），并使应用程序和全球数据服务摆脱对存储的依赖。
