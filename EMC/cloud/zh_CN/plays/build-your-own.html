实现 IT 即服务

自主构建

  • 2013 年，私有云基础架构支出将高达 123 亿美元，从现在直至 2017 年将以 18% 的速度增长 (IDC)
  • 到 2016 年，78% 的工作负载将在私有云中运行，IaaS 将以 42% 的速度增长（IDC、Gartner 和 EMC）

三大挑战

  1. IT 组织都在争夺存在“影子 IT”问题的客户

    终端用户和 LOB 巧妙地避开企业 IT，更倾向于采用内部无法提供的公共云产品。这种方法为每个应用程序的管理增加了风险、成本和复杂性， 并且违背了公司关于保护、安全性和可用性方面的标准。

    采用 EMC 和 VMware 技术实施基于 IaaS 的解决方案为 IT 奠定了坚实的基础，在灵活提供更多服务的同时，还可以维护企业标准。

  2. 提高效率，向公共云提供商看齐

    虽然许多组织已经实现了较高程度的虚拟化，但是他们并未高效地利用资源，并且其基础架构和虚拟服务器都存在资源过度调配的问题。他们认识到存在巨大的浪费，但是无法缩小差距。

    EMC 和 VMware 提供自动化生命周期管理、实时分析和历史报告，旨在充分地利用客户的资本支出和运营支出投资。

  3. IT 必须能够向企业展示价值

    IT 组织被企业视作成本中心，因此企业在管理数据中心时，一直在寻找更好地确定项目和投资优先事务的方式，从而减少预算资源。他们在寻找将资源消费者与运营支出和资本支出成本相关联的新方式。

    EMC 和 VMware IaaS 解决方案增强了对 IT 服务实际成本的可见性，可以将资源消耗与应用程序和基础架构所有者一一对应起来。

EMC 价值

  • 与 VMware 技术相集成的高度可扩展且可靠的存储平台，为私有云奠定了坚实的基础

    拥有最广泛的 VMware 集成的存储平台：

    • VAAI： 在阵列上处理更多工作，从而在服务器上运行更多 VM
    • VASA： 阵列可以与 VMware 进行通信，做到更加智能地使用存储
    • FAST： 根据应用程序需求实时优化工作负载
    • VSI： vCenter 插件支持 VMware 管理员调配存储并监视性能
    • ESA： 管理员可以在合并的 vCOPs 控制面板中查看阵列指标
    • UNISPHERE： 存储管理员可以查看 vSphere 环境
    • POWERPATH： 提高性能、可靠性和服务器密度
  • EMC 提供适用于私有云的集成保护和可用性解决方案，以减少风险并最大限度地增加应用程序正常运行时间

    凭借采用高级 VMware 集成和功能的业界领先的 EMC 备份，您可以轻松地在 VMware IaaS 云环境中以服务方式保护和存储数据，并提供客户所需的高级报告和可见性，确保其数据受到保护。

    RecoverPoint 提供本地和远程 vSphere 环境复制，支持强大的灾难恢复选项。

    借助 VPLEX，客户可以在多个阵列和站点之间分布 vSphere 环境，从而采用可以面向客户应用程序透明工作的主动-主动保护模式。

  • 支持客户自助访问符合企业 IT、安全性和治理标准的基础架构

    EMC 解决方案基于 VMware vCloud Automation Center 而构建，并使用大多数客户已经采用的 vSphere Suite。在该门户中，客户可以在不违反企业标准的情况下，轻松从集中式服务目录中为自己的 VM 调配资源以符合快速敏捷性要求。

    借助 EMC，客户可以更加轻松地管理 IaaS 云与物理硬件之间的集成。虚拟存储资源调配、自助备份和恢复以及安全性、法规遵从性和管理工具可以帮助 IT 提供更加可靠、一致的服务，同时又不会增加运营支出开销。

竞争

  • 凭借适用于所有存储平台的重复数据消除，通过统一存储体系结构和 ONTAP 操作系统提高效率
  • ONCOMMAND Insight 用于集成存储管理
  • 集群模式 ONTAP 文件系统支持全局命名空间和无中断数据移动
  • 可自定义的预先配置工作流库与 vCenter Orchestrator 相集成
  • 在任何资源级别均提供基于快照的精确数据保护

EMC 的回应

NetApp 销售存储和管理产品。所有其他云组件需要合作伙伴的产品拼凑而成。EMC 提供广泛的适用于云基础架构的产品组合： 存储层、复制平台、备份、外围环境/端点/访问保护，所有这些产品均可与 VMware 实现本机集成。如果客户打算使用 vCloud，EMC 可以加强其与 Vmware 的合作关系。

竞争对手推介

  • 利用预先配置的工作流自动化库，快速部署应用程序和基础架构，从而缩短收效时间
  • 企业用户可以利用自助式门户调配属于自己的资源，从而提高效率和用户满意度
  • 端到端高级生命周期管理
  • 利用单个管理控制台统一控制并减少复杂性

EMC 的回应

HP 在云领域的主要优势源于全面的基础架构堆栈： 捆绑式解决方案中的服务器、存储、虚拟机管理程序和管理层。在“自主构建”IaaS 销售策略中，EMC 提供可以与现有基础架构相集成的更加广泛的产品和解决方案组合。对于采用 VMware 技术的 IaaS 解决方案，EMC 提供了更高级别的集成。

竞争对手推介

  • 利用 AWS 基础架构的异地空间
  • 控制 IP 地址空间、网络和访问级别
  • 支持随时随地连接 — 企业网络、Internet、VPN
  • 成熟、安全、高度可用的云基础架构

注意： Amazon 目前只提供异地云托管服务。对于某些客户，他们可能会扩大服务范围，提供本地托管服务。

EMC 的回应

EMC 和 VMware 提供广泛的本地和混合云解决方案组合，支持客户构建适合用户的服务和 SLA，而不是简单地遵守宽泛的公共云标准。由于数据保留在内部，因此与安全性和法规遵从性相关联的风险大大降低。由于昂贵的数据迁移费用，公共云中的工作负载经常受到限制。

竞争对手推介

  • 灵活的基础架构，支持动态资源整合、快速的应用程序部署和弹性
  • 完整的云生命周期管理解决方案，快速设置、管理和支持企业云以及传统的 Oracle IT 环境
  • 内置的安全性和高可用性
  • 应用程序感知型虚拟化和管理功能，本机支持 Linux 和 Windows

EMC 的回应

Oracle 解决方案的大部分功能都是基于特定于 Oracle 的工作负载和产品： 服务器、存储、应用程序和虚拟机管理程序。虽然可以与 Oracle 数据库和应用程序工作负载本机集成，但是在基础架构/虚拟机管理程序堆栈中不支持第三方组件。EMC 支持 VMware 和更加开放的体系结构，因此客户可以采用自己的组件。

竞争对手推介

  • 完整的产品、服务和解决方案套件，专为混合平台而设计： 服务器（包括 x86 和 POWER）、刀片、存储和管理套件
  • 成熟的多层私有云使用情形支持： IaaS、DaaS、PaaS
  • 云服务标准化和自动化，采用可重复使用的工作负载模式
  • 接近瞬时的备份和恢复支持
  • 支持异构存储系统

EMC 的回应

SmartCloud 价值基于完整的 IBM 基础架构堆栈（服务器、存储及管理）和服务。为第三方组件提供有限的支持。客户在 IBM 专业服务方面遇到问题。EMC 依靠强大的合作伙伴和内部实施服务，提供自有本机安全性、复制和存储组件，并支持虚拟机管理程序和云软件。

竞争对手推介

  • 高性能，低成本
  • 整合资源模式向计算群集中的任何 VM 提供资源
  • 自助式管理控制台利用 VM 专用存储资源加快处理新工作负载
  • 随增长而付费的可扩展性支持增量添加新应用装置，以提高容量并为工作负载轻松地进行适当的资源调整
  • 不受虚拟机管理程序限制： 可以并行运行混合工作负载

EMC 的回应

Nutanix VCP 体系结构主要围绕基础架构虚拟化，并不是一个成熟的云计算平台。备份、异地复制和恢复以及安全性等重要组件需要由其他供应商提供。EMC 提供旨在满足所有这些云要求的自有产品，并且在虚拟机管理程序和云管理层提供卓越的集成。

竞争对手推介

  • 高性能、低成本，在 2U 应用装置中集成计算和存储
  • 随增长而付费的可扩展性支持增量添加新应用装置，以提高容量并为工作负载轻松地进行适当的资源调整
  • 凭借本机 SSD/HDD 分层、重复数据消除和数据压缩、面向 AWS 的集成复制以及高级备份功能，实现多面数据管理和保护

EMC 的回应

与 Nutanix 一样，Omnicube 也是注重基础架构虚拟化和性能。它的备份和异地复制功能比 Nutanix 更加先进，但是客户仍然需要提供来自其他供应商的备份硬件和安全功能。EMC 提供旨在满足所有这些云要求的自有产品，并且在虚拟机管理程序和云管理层提供卓越的集成。

资格确认问题

  • 副总裁： 您的团队明知数据中心内的大多数工作负载都存在资源过度调配的现象，但是应用程序所有者不愿放弃任何资源，是否因此而难以满足基础架构需求？

    EMC 和 VMware 解决方案可以报告基础架构每个层中的实际资源利用率。可以利用此类信息智能地发出警报，告知云消费者和管理员在哪些地方存在浪费资源的情况。然后，可以根据实际数据适当地调整应用程序工作负载，而不是靠凭空猜测。在按存储容量使用计费模式中恰当地实施，每个人都可以更加高效地利用回收的物理资源。

  • LOB： 您的 IT 团队的响应能力是否能够满足您对敏捷性和服务级别的要求？他们是否能够提供您运营业务所需的服务类型？

    通过自助式门户提供的自动化和流程编排功能是 EMC IaaS 解决方案的核心部分。通过使用服务目录，云消费者在选择从云中消费的服务时，能够获得与应用程序商店类似的体验。云管理员能够根据与企业安全性、法规遵从性和可用性策略保持一致的业务要求来构建服务集。

  • IT： 您的 IT 运营和支持团队是否具有相应的工具和流程从而可以主动采取行动，或者他们是否在不断地解决层出不穷的可用性和性能方面的问题？

    EMC 和 VMware IaaS 解决方案支持消费云资源的用户定义需要且愿意即付即用的服务级别。通过透明的按存储容量使用计费和按存储容量使用显示模式，企业能够选择最合适的服务，在满足工作负载需求的同时平衡业务价值。客户还应该考虑咨询和培训服务，从而快速启动转型。

异议处理

  • “我已经采用了 VMware，我可以从模板部署虚拟机，这是我自己的云。如果我的环境已经高度虚拟化，我为什么还需要 IaaS 呢？”

    高度虚拟化的数据中心可以奠定坚实的基础，但是 IaaS 添加了多租户、自助服务、计量/按存储容量使用计费、分析、自动化和流程编排等元素；虚拟化价值和优势可以超越 IT 界限，延伸到整个企业。在享受敏捷性的同时，满足应用程序工作负载的治理、控制和法规遵从性要求，支持新应用程序资源更加快速地收效，并直接为业务增加价值。

  • “现在 VMware 可以与我拥有的所有其他存储配合使用，我为什么还需要 EMC 呢？我们有许多供应商可选择。”

    正如 Wikibon 多年来在报告中指出的那样，EMC 可以提供与 VMware 的最佳集成。随着我们针对 VMware 的自动化和流程编排产品开发更多支持，该级别的集成将继续在我们的 IaaS 产品中提供。

    除了存储阵列，EMC 还通过提供旨在满足业务用户要求的最佳备份、复制、安全性和管理工具来完善 IaaS。

  • “我喜欢 IaaS 这个主意，听起来非常棒。但是似乎过于复杂，我们公司难以消受。”

    EMC 可以根据您公司的优先事务构建迈向云的路线图。在大多数客户环境中，IaaS 是最容易采用的云形式，并且最适合当前高度虚拟化的环境。我们的服务团队和 IaaS 参考体系结构可以帮助消除风险和复杂性。

    并非所有事情都需要一蹴而就，以渐进的方式向您的用户介绍 IaaS，会让您的组织快速成功，并且能够消除因为最终“追赶”进度而产生的较高成本。
