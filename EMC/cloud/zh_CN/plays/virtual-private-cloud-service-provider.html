实现 IT 即服务

托管私有云（服务提供商）

  • 客户通过可信赖的 EMC 服务提供商合作伙伴在私有云、托管私有云或公共云中分包特定的工作负载。帮助确定合适的合作伙伴并促进对话。

三大挑战

  1. 用于改善服务的 IT 预算有限

    企业要求 IT 做到事半功倍， 希望改善 IT 服务并提高性能， 还需要降低 IT 资本支出和运营支出，并更快速地实现营运资本回报。考虑到这些挑战，客户需要在评估哪些工作负载最适合异地部署时获得支持。

    EMC 服务提供商合作伙伴通过提供“即服务”选项优化工作负载，可以帮助客户降低成本，提高效率并增强应用程序可用性。

  2. 在当前的 IT 环境中敏捷性有限

    企业越来越希望 IT 能按需提供创新型解决方案。采用静态体系结构和陈旧流程时，由于更改起来非常复杂并且成本高昂，因此常常是避免更改。为了避免风险，普遍认可的观点是如果工作负载和应用程序保持在现场运行会更容易些。

    EMC 服务提供商合作伙伴提供许多“即服务”选项。私有云、托管私有云或公共云解决方案可以帮助 IT 部署工作负载并更快地实现价值。

  3. 失去控制，面临安全风险

    在将企业级应用程序迁移到异地时，安全问题是最大的顾虑。由于法规或性能要求，企业会限制移动工作负载，以便降低风险。因此，工作负载仍保留在 IT 基础架构中，而它们本可以通过外部模式获益 。

    EMC 服务提供商合作伙伴专门促进安全的工作负载迁移。可信赖的企业级专业技能可确保无缝过渡到云。

EMC 价值

  • 依靠 EMC 服务提供商合作伙伴，实现 IT 和业务线 (LoB) 应用程序高效转型

    EMC 与服务提供商合作伙伴携手，支持客户按需提供服务，并更加直观地了解 IT 按存储容量使用计费情况。将应用程序移到异地可以改善 SLA 和服务，同时减少所需的 IT 资源和支出。通过降低用于 IT 资产和基础架构的资本支出，并将这些支出变为运营支出预算，提高效率。同时还可以提高 IT 生产效率并节约更多成本，从而加快响应速度以及为企业提供服务的速度。通过 EMC 服务提供商合作伙伴提供的“即付即用”和“即服务”选项，可以优化资源。

  • 依靠 EMC 服务提供商合作伙伴，提高敏捷性和灵活性

    EMC 与服务提供商合作伙伴携手，为客户提供咨询服务，帮助他们建立敏捷、可扩展且经得起未来考验的 IT，从而获得竞争优势。依靠基于 EMC 技术的 EMC 服务，可以将应用程序和工作负载交由服务提供商管理。客户可以选择在私有云、托管私有云或公共云模式中部署、开发和迁移应用程序。EMC 可以提升服务提供商合作伙伴的能力，促进其技术、销售和市场营销团队发展，致力于合作共赢，而非相互竞争。EMC 提供了多种工具来确定最佳的工作负载放置和功能。

  • 依靠 EMC 服务提供商合作伙伴，获得信任、安全性和专业知识

    EMC 经验证的技术层产品组合与服务提供商专业知识配合使用，构成坚不可摧的强大组合。当迁移应用程序和工作负载时，客户需要进行轻松地迁移和连接，无论是在私有云、托管私有云还是公共云中。如今，法规遵从性、安全性和信任是优先考虑的事项。EMC 和我们的服务提供商合作伙伴提供顾问式评估，确保提供安全性、可用性和业务连续性以满足工作负载需求。

竞争

竞争对手推介

作为公共云的“先行者”，AWS 在基础架构即服务市场占据主导地位。技术创新、全面的服务、丰富的 API 和充满活力的开发人员也帮助 AWS 在平台即服务领域处于领先地位。AWS 采用具有竞争力的即付即用定价模式，而没有长期承诺。通过信用卡/礼品卡服务激活可以轻松启动服务，而不需要进行人工交互。

EMC 的回应

采用 EMC 技术的服务支持私有云、托管私有云和公共云模式，可为企业 IT 带来 AWS 无法提供的控制力和选择。通过 EMC 全球服务、我们的服务提供商合作伙伴和经销商，我们与 IT 专家建立了牢固的合作关系。我们针对有意义的 SLA、安全性、审核及法规遵从性和强大支持的重要性方面表现出了深入的理解能力，这为合作关系的建立奠定了基础。

竞争对手推介

Google Cloud Platform 刚刚进军 IaaS 市场领域，多年来主导搜索行业发展以及提供企业级高质量托管邮件和应用程序服务的过程中所积累的丰富经验为其奠定了坚实的基础。Google App Engine、Compute Engine、Cloud Storage 和 BigQuery 与 AWS 的同类产品展开竞争，但企业采用量有限。Google 采用极具竞争力的定价，并且支持使用信用卡轻松启动服务。

EMC 的回应

信任、信誉和经验是采用 EMC 技术的服务与 Google 云产品相比的竞争优势。EMC 支持将工作负载从数据中心移植到云中，并提供经验证的安全性、高可用性和领先的应用程序性能。依靠 EMC，IT 专家不必担心托管服务使用期限有限或 API 频繁更改等问题。我们可以提高性能和稳定性。

竞争对手推介

Microsoft 利用 Windows Azure 瞄准企业 IT，Windows Azure 是用于实现 IaaS、PaaS 和 SaaS 的公共云平台。MS 想要将企业从使用 Exchange 转换为使用 Office 365，通过支持以类似云的方式访问数据，降低许可证和 PC 维护成本。Azure 拥有极具竞争力的上市时间、灵活性和无下限的每分钟定价模式。对于 PaaS，Azure 支持在云中使用特定 MSDN 许可证。

EMC 的回应

灵活性高、选项多以及与大客户合作是采用 EMC 技术的云相较于 Azure 的优势所在。除支持 Hyper-V 外，我们还支持诸如 Linux 等非 Microsoft 虚拟化和操作系统环境。Microsoft 的云仅有少数几个“垄断”客户，如 Xbox 360 和 Bing。EMC 和我们的服务提供商及经销商合作伙伴为《财富》500 强中超过 99% 的公司和除此之外的全球数千家企业提供支持。

资格确认问题

  • 最高管理层级别高管、副总裁、LOB、IT： 您未来打算做哪些现在未完成的工作？是否打算改变业务或 IT 计划以影响更改并提高竞争力？

    这会引出关于通过 IT 转型实现战略性业务目标的讨论。要从最高管理层级别高管的思维角度进行沟通： 赢得市场份额、获得新的业务计划并利用 IT 提高利润。这让 EMC 和我们的服务提供商合作伙伴有机会向客户讲解选择合适的云模式获得竞争优势的业务影响。侧重于了解企业的愿景、方向和计划。

  • 首席财务官、首席信息官、IT： 您有 XaaS 方面的战略吗？您是否考虑将应用程序迁移到云？您如今是否依靠服务提供商将工作负载迁移到云？

    评估客户是否已经利用云选项。如果他们与 EMC 服务提供商合作伙伴合作来提供服务，那么这正是以现有信任为基础针对可能的云基础架构展开交流的好机会。如果客户正在研究“即服务”解决方案市场，那么向客户传达 EMC 服务提供商生态系统的价值。阐述 EMC 和服务提供商的关键竞争优势： 功能、专业知识、安全性和灵活性。

  • 最高管理层级别高管、副总裁、LOB、IT： 哪个云选项最能满足您的服务交付需求？哪些工作负载应该放置在哪里？您当前如何确保服务可用性？

    弄清楚组织是否真正了解云解决方案的益处？阐述“一刀切”方法不适合具有复杂性能和可用性需求的企业。向客户传达这样一个事实：EMC 和服务提供商提供广泛的存储、备份、分析、应用程序和安全性解决方案。服务提供商采用 EMC 技术提供解决方案，以确保在托管私有云中提高可用性和工作负载性能。

异议处理

  • “公共云中的数据容易受到黑客攻击以及发生其他安全问题。”

    依靠 EMC，您可以在私有云、托管私有云或公共云模式中获得信任和控制力。EMC 服务提供商合作伙伴可以提供与本地 IT 相媲美的安全性。EMC 和我们的合作伙伴提供的领先技术解决方案可以消除在将工作负载迁移到托管私有云时存在的风险。经验欠缺的云提供商在管理企业级应用程序方面缺乏行业经验，并且他们的解决方案通常无法满足法规要求。

  • “如果将服务迁移到云，那么 IT 部门将不再具有控制力。这对员工会产生负面影响。”

    EMC 和服务提供商提供的解决方案使业务线和 IT 职能部门都能受益。当 IT 提供卓越的服务时，将有助于将自身形象提升为关键业务合作伙伴。通过迁移操作任务和工作负载并确定其优先顺序，可以将资源转移到创新型战略项目上。优化资源可以节省成本并提高效率。IT 部门和专家的专业技能可以得到进一步利用，从而实现业务目标。

  • “Amazon 和 Google 成本较低并且是‘即服务’领域的领导者。”

    “尽管 Amazon 和 Google 提供预打包的解决方案，并且支持信用卡结账，但随时间推移真实成本的透明度引质疑。此外，安全性、性能和可移植性问题令人担忧。EMC 技术与服务提供商专业知识相结合，作为一种替代方案，可以提供企业级解决方案。商品化公共云提供商仅限于提供多租户解决方案，而 EMC 服务提供商可以提供私有云、托管私有云和公共云混合环境。”
